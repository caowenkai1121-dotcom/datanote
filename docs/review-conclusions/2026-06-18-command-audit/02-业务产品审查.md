# 02-业务产品审查

## 业务闭环判断

【Domain】一个真正可用的数据中台，核心闭环应是：

1. 接入数据源。
2. 采集元数据。
3. 在数据地图识别资产、分级、血缘和质量。
4. 把问题升级为治理工单或项目任务。
5. 通过开发、同步、建模、主数据等模块产生变更。
6. 走审批和发布。
7. 调度执行并监控。
8. 审计留痕并回到治理看板。

当前系统多数模块都有实现，但用户路径仍被三类问题割裂：未完成入口、隐藏深链、权限/错误不解释。

## 产品原则

【Product】后续实施必须遵守：

- 没有真实场景的功能，删除或隐藏。
- 有真实场景但低频的功能，收进主流程的“更多/配置”入口。
- 对安全高风险但业务必要的功能，增加权限、审批、二次确认和审计，而不是随意开放。
- 对用户不理解的失败，页面要给下一步，不要只报技术错误。

## 无意义功能处理清单

### 1. “即将完善”占位必须下架

证据：

- `src/main/resources/static/js/project.js:651`。

【Domain】真实业务里用户点进项目页，是要看成员、资产、任务、发布、活动、健康，而不是看到“即将完善”。这会降低用户对系统可信度。

【Product】Claude 必须判断该 tab 属于哪一种：

- 若没有业务价值：删除入口。
- 若与已有 tab 重复：合并到已有 tab。
- 若确有价值：补齐最小闭环并写清空态文案。

### 2. hidden 模块不能成为幽灵页面

证据：

- `src/main/resources/static/workspace.html:7530` 治理主题域被 hidden 但保留深链。
- `src/main/resources/static/workspace.html:7591` 主数据质量、存活性、层级管理被 hidden 但保留深链。
- `src/main/resources/static/workspace.html:7580` hidden 治理模块仍可 `switchGovModule`。
- `src/main/resources/static/workspace.html:7646` hidden MDM 模块仍可进入。

【Domain】低频模块隐藏是合理的，但不能让用户从链接进入后找不到当前位置。

【Product】Claude 需要建立显式去向：

- `subject`：从资产目录或治理配置进入，并显示“该能力为资产主题配置”。
- `quality`：并入数据管家待办或质量工单。
- `survivorship`：并入匹配去重的子配置。
- `hierarchy`：若场景弱，默认隐藏并提示归并入口。

## 用户体验流程断点

### 1. 权限失败缺少业务解释

【Domain】实际企业中，普通用户没有权限是正常的，但系统必须告诉他“缺什么权限”和“找谁处理”。

证据：

- `src/main/resources/static/workspace.html:7805` 只提示“无权访问该模块”。

【Product】后续无权限页必须包含：

- 当前账号。
- 访问的模块或操作。
- 缺失权限点。
- 推荐申请对象：项目管理员、数据管理员、系统管理员。

### 2. 登录/开放模式不够清晰

【Domain】测试环境或开放模式下，用户需要明确入口，不应猜测是否能进。

证据：

- `src/main/resources/static/login.html:181` 登录页只请求状态，缺少明确失败/开放入口处理。

【Product】开放模式显示“进入工作台”，失败显示“状态加载失败，重试”。

### 3. 主数据和治理的业务动作要更少但更清楚

【Domain】MDM、质量、存活性、层级、交叉引用、订阅发布都有真实场景，但不应该都成为同级主菜单。中台用户更需要“下一步要处理什么”。

【Product】保留数据管家工作台作为聚合入口：

- 不合规黄金记录 -> 升级治理工单。
- 重复簇 -> 去重合并。
- 存活规则缺失 -> 从去重页进入配置。
- 交叉引用缺失 -> 从黄金记录详情进入。

## 高风险功能的产品化要求

### 1. SQL、DDL、Shell 不是普通按钮

证据：

- `src/main/java/com/datanote/platform/iam/PermInterceptor.java:94` SQL/元数据中心相关权限仍偏粗。
- `src/main/java/com/datanote/domain/orchestration/TaskExecutionService.java:64` Shell 默认关闭。
- `src/main/java/com/datanote/domain/orchestration/TaskExecutionService.java:388` 开启后执行 `/bin/bash -c`。

【Domain】真实生产中，SQL DDL、Shell 都属于高风险操作。用户可以有能力，但系统必须提醒代价。

【Product】后续要求：

- SELECT/WITH、DDL、DML、危险 SQL 分级。
- DDL/DML 至少二次确认，高危语句走审批。
- Shell 类型生产默认不可用；如果启用，必须单独权限、双人审批和审计。

### 2. AI 取样例数据要按“真实数据访问”看待

证据：

- `src/main/java/com/datanote/platform/ai/agent/tool/impl/TableDataTool.java:16` 工具说明会查看真实样例数据。
- `src/main/java/com/datanote/platform/ai/agent/tool/impl/TableDataTool.java:54` 调用 `sampleRows`。
- `src/main/java/com/datanote/domain/governance/AssetDetailService.java:179` 提供样例读取。

【Domain】AI 不是特殊用户，不能绕过数据权限。用户问“看几行数据”时，本质是在访问真实数据。

【Product】AI 工具展示真实数据前必须沿用表级 ACL、列脱敏、行过滤、limit 和审计。

## 保留、合并、下架建议

【Product】保留：

- 数据地图、治理总览、质量工单、项目空间、数据同步、数据开发、主数据黄金记录、审计、AI 助手。

【Product】合并：

- 主数据质量监控并入数据管家待办。
- 存活性规则并入去重合并配置。
- 主题域作为资产目录/治理配置入口，不做一等菜单。
- 项目资产健康、发布检查、任务待办应在项目详情内形成一个闭环。

【Product】下架或隐藏：

- 只有“即将完善”的入口。
- 没有数据来源、没有操作结果、没有验收标准的页面。
- 与已有模块职责重复但没有新增价值的入口。
