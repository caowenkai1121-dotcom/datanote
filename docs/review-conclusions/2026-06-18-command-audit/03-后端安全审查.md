# 03-后端安全审查

## 总体结论

【Dev-A】默认认证开启时，本轮未发现可直接匿名打穿系统的 P0。但系统仍存在多处高优先级安全边界问题，尤其是 Session 场景 CSRF、AI 审批归属、Shell 执行、SQL 权限粒度、公共 WebSocket topic、旧 DataX/DDL 旁路和 DataX 临时明文文件。

【PM-Lead】安全项不能只写进文档，必须由 Claude 分批修复并补测试。

## P0：公共 WebSocket topic 存在跨用户可见风险

证据：

- `src/main/java/com/datanote/common/LogBroadcastService.java:40` 推送 `/topic/task-log`。
- `src/main/java/com/datanote/common/LogBroadcastService.java:58` 推送 `/topic/task-status`。
- `src/main/java/com/datanote/common/LogBroadcastService.java:88` 推送 `/topic/notification`。
- `src/main/java/com/datanote/common/LogBroadcastService.java:98` 使用 `convertAndSend`。
- `src/main/java/com/datanote/domain/integration/HiveDdlController.java:503` 推送 `/topic/sql-log`。
- `src/main/resources/static/workspace.html:17635` 到 `17653` 前端订阅公共 topic。

影响：

- 任务日志、任务状态、通知、SQL 执行日志可能被非当前用户看到。

Claude 修复要求：

- 迁到 `/user/queue/*` 或包含 owner/session/jobId 的私有 topic。
- 服务端推送必须绑定当前用户、任务归属或会话归属。
- 前端只订阅当前用户和当前任务相关通道。
- 补跨用户订阅不可见测试或至少写可复现实验。

## P0：旧 DataX / Hive / Doris DDL 入口绕过新同步域

证据：

- `src/main/java/com/datanote/domain/integration/DataxController.java:93` `run` 按 `jobId` 运行。
- `src/main/java/com/datanote/domain/integration/DataxController.java:124` 仍有创建并同步入口。
- `src/main/java/com/datanote/domain/integration/service/SyncJobService.java:87` 新同步域已有 `requireDatasourceAccess`。

影响：

- 旧入口和新同步域的任务归属、数据源权限、审计策略不一致。

Claude 修复要求：

- 对 `/api/datax/*`、Hive/Doris DDL 类入口统一引入数据源访问校验。
- job registry 至少存 `jobId/userId/sourceDsId/targetDsId/syncTaskId/expiresAt`。
- 运行时逐项校验归属和权限。

## P0：SQL SSE executionId 未绑定提交用户

证据：

- `src/main/java/com/datanote/domain/integration/HiveDdlController.java:235` `pendingSql` 只存 `Long -> String`。
- `src/main/java/com/datanote/domain/integration/HiveDdlController.java:263` 写入 pending SQL。
- `src/main/java/com/datanote/domain/integration/HiveDdlController.java:284` 只按 executionId 判断。

影响：

- executionId 是 SQL 执行会话资源，应绑定发起人，否则可能跨用户消费或观察。

Claude 修复要求：

- pending 值改为包含 `sql/ownerUserId/createdAt/permissionSnapshot`。
- SSE 建连时校验当前用户。
- 使用原子 remove，增加 TTL 清理。

## P1：Session 认证下关闭 CSRF

证据：

- `src/main/java/com/datanote/platform/config/SecurityConfig.java:84` `http.csrf().disable()`。
- `src/main/resources/application.yml:9` 到 `13` 为认证相关配置。

影响：

- 项目使用 Cookie/Session 认证时，状态变更接口需要 CSRF 保护。

Claude 修复要求：

- 启用 Spring Security CSRF 或自定义防 CSRF header。
- 仅对明确无状态接口 ignore。
- 补 MockMvc：无 token 的 POST/PUT/DELETE 返回 403。

## P1：AI Agent 审批决定缺少会话归属校验

证据：

- `src/main/java/com/datanote/platform/ai/agent/web/AiAgentController.java:167` 到 `187` 审批决定逻辑。

影响：

- 知道审批 ID 的有权限用户可能处理他人 Agent 审批。

Claude 修复要求：

- 读取审批对应 session 后复用 session owner 校验。
- 更新时同时加 `id/status/session_id` 条件。
- 补 direct-ID 越权审批测试。

## P1：Shell 脚本执行开启后风险极高

证据：

- `src/main/java/com/datanote/domain/orchestration/TaskExecutionService.java:64` `datanote.task.shell-enabled` 默认 false。
- `src/main/java/com/datanote/domain/orchestration/TaskExecutionService.java:388` 执行 `/bin/bash -c executeSql`。

影响：

- 一旦开启，就是任意命令执行面。

Claude 修复要求：

- 生产环境禁用或删除 Shell 类型。
- 如必须保留，独立权限点 `develop:shell-run`。
- 低权限沙箱/容器执行，清空环境变量，限制工作目录。
- 双人审批，任何人包括 admin 不可自审。

## P1：SQL 执行权限粒度过粗

证据：

- `src/main/java/com/datanote/platform/iam/PermInterceptor.java:94` 元数据中心相关接口仍映射到 `catalog:edit`。
- `src/main/java/com/datanote/domain/integration/HiveDdlController.java:70` 起包含 DDL 类接口。

影响：

- `catalog:edit` 语义过宽，不应等同任意 SQL/DDL/DML 执行权。

Claude 修复要求：

- 拆分 `sql:read`、`sql:execute`、`sql:ddl`、`sql:danger`。
- 默认只允许 SELECT/WITH。
- DROP/TRUNCATE/GRANT 等危险语句默认拒绝或高危审批。

## P1：DataX 临时明文 JSON 异常路径清理不完整

证据：

- `src/main/java/com/datanote/domain/integration/DataxService.java:22` DataX 服务入口。
- `src/main/java/com/datanote/domain/integration/DataxController.java:124` 创建并执行链路。
- `src/main/java/com/datanote/domain/orchestration/TaskExecutionService.java:312` 到 `325` 调度执行 DataX 相关链路。

影响：

- DataX job 文件包含源库和目标库密码，异常时残留会造成凭据暴露。

Claude 修复要求：

- `generateJobJson -> runJob` 必须包在 `try/finally deleteIfExists`。
- 删除失败记录告警但不能输出明文密码。
- 补测试模拟 `runJob` 抛异常后文件仍删除。

## P2：其他安全收敛项

【Dev-A】后续还需处理：

- `src/main/java/com/datanote/platform/iam/PermInterceptor.java:155` 未映射写接口默认只需登录，应逐步改为默认拒绝。
- `src/main/java/com/datanote/platform/iam/DataAclService.java:44`、`106` `resourceId == null` 直接放行，应对敏感资源 fail-closed。
- `src/main/java/com/datanote/platform/portal/DashboardController.java:128`、`140` 运维服务/指标对任意登录用户可见，应加 `operations:monitor` 或等价权限。
- `src/main/java/com/datanote/platform/audit/AuditService.java:160` 到 `169` 审计 detail 入库前应统一脱敏。
- `src/main/java/com/datanote/domain/develop/ScriptApprovalService.java:74` 到 `76` admin 自审应禁止。
- `src/main/java/com/datanote/platform/ai/agent/web/AiAgentController.java:85` 到 `98` AI SSE 不应允许未知 sessionId 预注册。
- `src/main/java/com/datanote/platform/ai/agent/web/AiFileController.java:78` 到 `85` 文件下载 Content-Type 应按白名单或统一 octet-stream。
