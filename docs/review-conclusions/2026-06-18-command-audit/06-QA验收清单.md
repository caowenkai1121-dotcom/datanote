# 06-QA 验收清单

## 总原则

【QA】Claude 修复后，不能只看代码“像是改了”。必须用命令、浏览器行为、权限场景和异常场景证明问题闭环。

本清单用于修复后回归。当前阶段不部署。

## 基础命令

每批后端改动后至少运行：

```powershell
mvn -q test
```

涉及打包或配置时运行：

```powershell
mvn -q -DskipTests package
```

涉及前端 JS 文件时，对改动文件运行：

```powershell
node --check src/main/resources/static/workspace.html
node --check src/main/resources/static/js/project.js
```

如果 `workspace.html` 无法直接用 `node --check`，应抽取或使用现有项目惯例校验 `<script>`，并在结果里说明。

## P0 验收

### 启动鉴权

- `/api/auth/status` 失败：不能进入工作台。
- `/api/rbac/me` 失败：不能进入工作台。
- 开放模式：有明确“进入工作台”路径。
- 登录态正常：能进入首页。

### WebSocket 私有化

- 用户 A 启动任务，用户 B 不收到 A 的任务日志。
- SQL 日志只出现在执行者会话。
- 通知只发给目标用户或目标角色。

### DataX/DDL 旧入口

- 无数据源权限时，旧 DataX `run` 拒绝。
- jobId 过期、非本人、非授权任务均拒绝。
- 拒绝响应不泄露密码、连接串、服务器路径。

### SQL SSE

- 用户 A 创建 executionId，用户 B 消费失败。
- 同一个 executionId 第二次消费失败。
- TTL 过期后消费失败。

### AI 样例数据

- AI `table_data` 对无权限表拒绝。
- 有脱敏策略的列返回脱敏值。
- 行级权限对 AI 样例同样生效。

### 无意义入口

- UI 中不再出现“即将完善”。
- 被隐藏或合并的入口有明确跳转或说明。

## P1 验收

### CSRF

- 未带 token 的 POST/PUT/DELETE 返回 403。
- 正常页面请求能自动带 token 并成功。
- 明确无状态接口有白名单和测试。

### AI 审批

- 非 session owner 不能审批他人 Agent 审批。
- owner 或有明确授权的人可正常处理。
- 重复审批不会成功。

### Shell

- 默认配置下 Shell 不可执行。
- 开启后仍需要独立权限。
- admin 不可自审。

### SQL 权限

- 只有 `sql:read` 的用户只能 SELECT/WITH。
- DDL/DML 需要更高权限或审批。
- DROP/TRUNCATE/GRANT 默认拒绝。

### 弹层和权限体验

- 旧 `confirmBox/promptBox/showModal` 主要调用点不再作为主路径。
- 弹层有可访问语义和焦点行为。
- 无权限深链显示缺失权限点，不只 toast。
- 小屏顶栏不溢出，关键入口可用。

## UI 人性化检查

【QA】每个被改页面至少检查：

- 加载中：有骨架或明确 loading。
- 空态：说明为什么空，下一步做什么。
- 错误态：说明失败原因，并有重试。
- 禁用态：说明为什么不可点。
- 破坏性操作：二次确认文案包含对象名和后果。
- 长文本：不撑破布局，可悬停或查看详情。
- 小屏：375px、768px、1366px 三个宽度不重叠。

## 业务回归路径

【QA】至少手测这些主路径：

1. 登录/开放模式进入首页。
2. 数据源列表 -> 元数据采集 -> 数据地图搜索 -> 表详情。
3. 表详情 -> 质量/血缘/治理动作。
4. 治理问题 -> 升级项目任务。
5. 项目空间 -> 成员/资产/任务/发布检查。
6. 数据同步任务 -> 创建 -> 运行 -> 日志 -> 详情抽屉。
7. AI 助手 -> 搜资产 -> 请求样例数据 -> 权限/脱敏生效。
8. 审计中心 -> 能查到关键写操作但无敏感明文。

## 不放行条件

出现任一情况，QA 直接打回：

- 权限接口失败仍进入工作台。
- 任意跨用户日志、SQL、通知可见。
- 真实数据绕过 ACL 或脱敏。
- 页面继续出现“即将完善”。
- 无权限只有 toast，没有缺失权限说明。
- Shell 在默认配置下可执行。
- 修复引入新依赖或大面积格式化。
- 测试未跑，或失败但未解释。
