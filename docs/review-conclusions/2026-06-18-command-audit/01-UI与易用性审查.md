# 01-UI 与易用性审查

## 总体结论

【Design】当前 UI 已有统一设计规范和 `DN.*` 公共组件，但历史页面仍有大量旧弹层、内联样式、手写 `innerHTML`、小屏不友好布局和隐藏模块深链。用户会感到“功能很多，但入口解释不足、失败原因不清楚、部分页面像未完成”。

【Domain】中台用户不是来欣赏界面的，他们需要稳定完成工作：看数据、处理工单、建任务、发布、审计。所有界面都应减少疑惑，失败时告诉用户下一步。

## P0：启动鉴权失败不能进入工作台

【Design】`workspace.html` 的 `dnBootGate` 在认证或权限接口失败时仍可能进入工作台。

证据：

- `src/main/resources/static/workspace.html:14036` 定义 `dnBootGate`。
- `src/main/resources/static/workspace.html:14037` 请求 `/api/auth/status`。
- `src/main/resources/static/workspace.html:14042` 请求 `/api/rbac/me`。
- `src/main/resources/static/workspace.html:7802` 路由守卫依赖 `__authEnabled`。

问题：

- 权限加载失败时继续渲染，会让菜单、按钮、深链守卫处于不可信状态。
- 对用户来说就是“有时能进、有时没权限、有时按钮消失”，不可解释。

Claude 修复要求：

- 把启动状态改成 `loading / ready / auth-error`。
- 只有开放模式，或“已认证且 RBAC 成功”才能进入 `navigateTo`。
- `/api/auth/status` 或 `/api/rbac/me` 失败时渲染全屏错误态，提供重试。
- 不要用 toast 一闪而过代替启动失败页。

## P1：弹层体系必须统一并具备可访问性

【Design】项目中仍存在 `showModal`、`confirmBox`、`promptBox` 与 `DN.confirm`、`DN.drawer` 并存。

证据：

- `src/main/resources/static/workspace.html:5443` 自定义 `showModal`。
- `src/main/resources/static/workspace.html:5491` 自定义 `confirmBox`。
- `src/main/resources/static/workspace.html:5499` 自定义 `promptBox`。
- `src/main/resources/static/workspace.html:2975`、`3005`、`3030`、`3055` 使用 `promptBox`。
- `src/main/resources/static/workspace.html:3067`、`6204`、`15477`、`15982` 使用 `confirmBox`。
- `docs/设计规范.md:163` 禁止原生确认/输入范式，要求用 `DN.confirm`。
- `docs/设计规范.md:208` 要求弹层遵守统一纪律。

问题：

- 旧弹层焦点管理、ARIA、层级、错误状态不统一。
- 抽屉内再开旧弹层容易出现遮罩、焦点、z-index 混乱。
- 输入型 prompt 对复杂表单不友好，也不便于校验错误。

Claude 修复要求：

- 破坏性确认全部迁到 `DN.confirm`。
- 输入表单迁到 `DN.drawer` 或统一 `DN.modal` 表单。
- 弹层必须有 `role="dialog"`、`aria-modal="true"`、焦点进入和返回。
- 每次只迁一组调用点，避免一次性重写整段工作台。

## P1：权限不足要可解释

【Design】当前无权限菜单和按钮多为隐藏，深链只提示“无权访问该模块”。

证据：

- `src/main/resources/static/workspace.html:7805` 仅 toast “无权访问该模块”。
- `src/main/resources/static/workspace.html:13877`、`13885`、`13893` 依赖权限隐藏/观察器。
- `src/main/resources/static/workspace.html:12017` 命令面板过滤权限项。

问题：

- 用户不知道缺哪个权限点。
- 用户不知道该找谁申请。
- 管理员排查权限时缺少自检入口。

Claude 修复要求：

- 模块深链无权限时展示专门无权限页，至少包含当前用户、目标模块、缺失权限点、建议联系角色。
- 命令面板可保留禁用态条目，并显示“缺少 xxx 权限”，不要全部消失。
- 账号抽屉增加“我的权限”或“权限概览”入口。

## P1：小屏顶栏需要收纳

【Design】顶栏右侧工具区过长，小屏容易挤压。

证据：

- `src/main/resources/static/workspace.html:110` 顶栏右侧工具区开始。
- `src/main/resources/static/css/app.css:112` `.header-right` 固定不收缩。

Claude 修复要求：

- `max-width:768px` 下只保留命令面板和用户入口。
- 帮助、通知、主题、密度、AI 等放入“更多”菜单。
- 环境标识在小屏可隐藏或压缩为短标识。

## P1：登录开放模式要给直接入口

【Design】登录页开放模式只提示“可直接进入”，但用户没有明确按钮；状态接口失败也不够显性。

证据：

- `src/main/resources/static/login.html:181` 请求 `/api/auth/status`。

Claude 修复要求：

- 当 `authEnabled === false` 时展示“进入工作台”主按钮，跳转 `workspace.html#/home`。
- 状态接口失败时展示可重试错误，不要静默。

## P1：删除无完成定义的占位功能

【Product】`project.js` 仍存在“即将完善”占位，这是典型无意义功能入口。

证据：

- `src/main/resources/static/js/project.js:651` 未实现 tab 直接写入“即将完善”。

Claude 修复要求：

- 如果该 tab 没有真实业务动作，隐藏入口。
- 如果必须保留，补齐最小可用闭环：数据来源、操作、空态、错误态、验收标准。
- 禁止继续显示“即将完善”。

## P2：视觉系统继续收敛

【Design】仍有大量内联样式和手写 HTML。不能一次性清完，但新增和触碰代码必须逐步迁移。

证据：

- `src/main/resources/static/workspace.html:45` 仍有 body 内 `<style>`。
- `src/main/resources/static/js/project.js:17`、`256`、`287`、`592` 等存在手写样式和打印样式。
- `src/main/resources/static/css/app.css:3263` 同步详情指标网格固定 `repeat(3, 1fr)`。
- `src/main/resources/static/css/app.css:3281` KV 固定 `120px 1fr`。
- `docs/设计规范.md:171` 要求空态使用 `DN.empty`，禁 emoji。
- `docs/设计规范.md:172` 要求错误态使用 `DN.errorBox`。

Claude 修复要求：

- 触碰到的 UI 片段优先换 `DN.h`、`DN.empty`、`DN.errorBox`、`DN.confirm`。
- 内联 `style` 优先迁到模块类，颜色/间距走 CSS 令牌。
- 同步详情小屏补断点：640px 两列，420px 单列，KV 小屏改单列。
- 命令面板和图标不要继续使用 emoji 作为功能图标。
