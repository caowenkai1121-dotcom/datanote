# DataNote 设计规范 — "Slate & Indigo / 石墨靛蓝"

> 适用范围:DataNote 全栈单体平台前端(workspace.html 单文件 SPA + gov-*/mdm-* 模块)。
> 落地原则:**只换皮、不动骨**——不改任何类名 / id / DOM / 事件绑定,仅通过 CSS 变量与组件样式重塑视觉。
> 版本:v2.0(全新视觉基线)。

---

## 1. 设计理念

| 关键词 | 含义 |
|---|---|
| 克制专业 | 工程平台气质,降噪优先,主色用量节制,不堆渐变与重阴影 |
| 层级清晰 | 信息密集场景靠灰阶 + 边框 + 间距立层级,而非靠颜色堆叠 |
| 高可读 | 正文对比度全部 ≥ 4.5:1(WCAG AA),数字 tabular-nums 对齐 |
| 体系化 | 颜色/圆角/阴影/间距/动效全部令牌化,杜绝随手硬编码 |
| 双主题 | 同一套令牌驱动亮/暗两套主题,`[data-theme="dark"]` 一键切换 |

## 2. 色彩系统

### 2.1 品牌主色(Indigo / 靛蓝)
由旧的 Ant 天蓝 `#1890ff` 迁移到更沉稳专业的石墨靛蓝。

| 令牌 | 亮色值 | 用途 |
|---|---|---|
| `--primary` | `#3457d5` | 品牌色 / 主按钮 / 链接 / 激活态 / 数据强调 |
| `--primary-hover` | `#4f6ef0` | 主按钮、可交互元素 hover |
| `--primary-light` | `rgba(52,87,213,.09)` | 浅底:hover 背景、tab 激活底、图标容器底 |
| `--primary-bg` | `rgba(52,87,213,.08)` | 行选中 / 日期选中 / 表格 hover 统一底 |

> 主色对白字对比度 4.9:1(AA 通过,适合按钮文字)。

## 3. 圆角 · 阴影 · 焦点环

### 3.1 圆角(4 档体系)
| 令牌 | 值 | 用途 |
|---|---|---|
| `--radius-sm` | `4px` | 标签 / 小徽标 / 图标按钮 |
| `--radius` | `6px` | 按钮 / 输入框 / 下拉(原 3px → 6px) |
| `--radius-lg` | `10px` | 卡片 / 弹窗 / 抽屉 |
| `--radius-xl` | `14px` | 大图标容器 / 大卡 |
| 全圆 | `999px` | 药丸 / 头像 / 开关 |

### 3.2 阴影(3 档,中性偏冷)
| 令牌 | 值 | 用途 |
|---|---|---|
| `--shadow-sm` / `--shadow` | `0 1px 2px rgba(20,28,58,.06)` | 默认轻浮 |
| `--shadow-md` | `0 4px 14px rgba(20,28,58,.08)` | 卡片 hover / 下拉浮层 |
| `--shadow-lg` | `0 12px 40px rgba(15,20,40,.16)` | 弹窗 / 抽屉 |

### 3.3 焦点环(无障碍)
- `--ring`: `0 0 0 3px rgba(52,87,213,.28)`(输入聚焦)
- 键盘焦点:`outline:2px solid var(--primary); outline-offset:2px`(沿用现有 focus-visible 规则,仅换色)

## 10. 工程强制规则(开发任何新页面 / 新 UI 组件必读)

> 本设计系统**全局生效**。此后所有新增页面、模块、UI 组件**必须**遵守本规范与下列硬规则;不得另起炉灶自定义视觉。代码评审以此为门禁。

### 10.1 优先用组件工厂(dn-common.js 的 `DN.*`)
新 UI 一律先用现成工厂,它们已内建本设计系统,不要手写裸 DOM + 内联样式重造轮子:
`DN.h`(建元素)/ `DN.card`(标准卡:头+体)/ `DN.statTile`(KPI 磁贴)/ `DN.pill`(状态药丸)/ `DN.dbTablePicker`(库表选择器)/ `DN.icon`(图标)/ `DN.drawer`(右滑抽屉)/ `DN.toast`(提示)/ 表格分页组件。治理/主数据模块再叠加 `gov-*` 类(须渲染进 `#govModuleContent`/`#mdmModuleContent`/`#homeCockpit` 容器内,否则 `gov-*`/`dash-*` 作用域令牌不生效)。

### 10.2 颜色 / 间距 / 圆角 / 阴影 一律走令牌, 禁硬编码
- 颜色**禁**写死 `#1890ff`/`#52c41a`/`#888` 等,一律 `var(--primary)` / `var(--success)` / `var(--text-muted)` …(完整令牌见 §2)。需要兜底就 `var(--token, 合理回退)`。
- 圆角用 `var(--radius*)`(§3.1),阴影用 `var(--shadow*)`(§3.2),间距用 4px 网格(§5)。不要各页各写魔法 px。

### 10.3 CSS 变量「雷」铁律(踩过两次, 全站主按钮 / 卡片曾因此隐形 / 发扁)
- **局部作用域内才定义的变量**(如 `--dash-*` 只在 `:is(#homeCockpit,#govModuleContent,#mdmModuleContent)` 内、`--gov-*` 同理)**绝不能被全局(无 `#id` 限定)规则直接 `var()` 引用且无 fallback**——容器外该变量未定义,会让**整条属性失效**(背景透明→元素隐形、阴影丢失→发扁)。
- 规则:① 跨容器共享的令牌放 `:root` 全局定义;② 或在引用处带可视 fallback `var(--x, 实值)`;③ 注意**嵌套 fallback 是生效的**(`var(--a, var(--b))` 合法,b 会兜底)。
- 自查:写全局 `.xxx{ ...: var(--y) }` 前,确认 `--y` 在 `:root` 有定义。

### 10.4 布局 / 文本 防错位铁律
- **inline-flex 行内的 `<select>`/`<input>`**:在 flex 横排里要 `width:auto`(配 `min-width`),**禁 `width:100%`**(会占满整行把同行元素挤到换行竖排)。栈式表单(标签上/左、控件独占一行)才用 `width:100%`。
- **行内徽标 / 药丸 / 标签**:必须 `white-space:nowrap`,否则窄列里中文逐字竖排(客\n户)。
- **表格里的长编码 / 长文本列**:`display:inline-block; max-width:Npx; white-space:nowrap; overflow:hidden; text-overflow:ellipsis`(+`title` 全文),不要让它撑破表格或竖排。
- **等宽卡片网格**:短内容卡用 `align-items:start`(不强制等高拉伸出空白);确需等高再 `stretch`。
- **图标**:`DN.icon` 输出已 `width/height=100%` 填满容器盒——把它放进**有明确宽高的容器**,别放无尺寸容器(否则撑大)。

### 10.5 状态齐全(交互必备)
- 每个可点元素都要有 `hover` + `:focus-visible`(2px 主色 outline)+ `:active` + `disabled`(变灰)反馈。
- 列表 / 表格要有**空态**占位;加载用骨架;**错误内联提示**,禁 `alert()`。
- 表单要有校验与内联错误;危险操作二次确认;提交中锁按钮防重复。

### 10.6 改静态资源后的发布动作
改 `css/*.css` 或 `js/*.js` 后:bump `workspace.html` 里对应 `?v=` 版本号 → 重打包重启 → 浏览器 `?nocache=x` 强刷验证。**改 `workspace.html` 用编辑器精确改,勿整文件重写(会规范化行尾致 diff 爆炸)**。
