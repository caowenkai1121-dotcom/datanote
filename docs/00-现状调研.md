# DataNote 现状调研

> 调研时间：2026-06-18  
> 范围：本地全栈单体代码、静态前端、权限拦截、配置与部署脚本、既有项目文档。  
> 原则：只记录有代码或文档证据的问题，不写入任何远程凭据。

## 一、总体判断

【PM-Lead】系统已经形成“元数据、治理、主数据、同步、项目、AI 助手、系统管理”一体化中台雏形，但仍有三类核心风险：安全边界不够收敛、产品/权限/导航口径不一致、前端设计系统未完全统一。当前最适合按轮次推进：先封高风险入口，再收敛模块边界，最后补齐业务闭环和 UI 统一。

## 二、多角色审查结论

### 1. 业务流程

【Domain】表级数据权限只覆盖部分入口，预览、SQL、治理资产、MDM 导入等读数链路仍可能绕过统一 ACL。

【Domain】项目角色矩阵仍偏“软控制”，成员、任务、资产、发布等核心操作需要在 service 层继续强化项目内角色校验。

【Domain】部分操作者字段信任请求参数，指标计算与 MDM 导入已改为服务端当前登录用户；其他类似入口后续继续排查。

### 2. 产品与信息架构

【Product】“数据同步 / 数据集成 / 数据运维”的导航归属、路由守卫和权限点仍未完全统一。

【Product】前端曾使用不存在的权限点 `metrics:run`、`project:edit`，已改为现有权限口径。

【Product】`docs/02-信息架构v3.md` 与真实页面存在偏差，后续需要把“数据模型”“数据同步并入运维”“首页卡片数”同步成唯一契约。

### 3. 架构

【Architect】配置来源分裂，历史上存在代码默认值、配置文件、环境变量、数据库配置、部署脚本共同生效的问题。本轮已移除 Doris/部署链路中的可上线默认连接值和明文痕迹，后续仍需建立统一配置契约。

【Architect】Controller 直接暴露实体或 Map 的情况较多，前后端契约缺少稳定 DTO。后续优先从高频 API 建 Request/Response DTO 与统一 API client。

【Architect】跨域联动多为直接调用，缺少领域事件或应用服务编排层。后续先覆盖“模型发布落资产”“质量问题转项目任务”两条链路。

### 4. 后端安全与稳定性

【Dev-A】认证原先由“密码非空”隐式决定是否启用，存在未配置即全站放行风险。本轮改为 `DATANOTE_AUTH_ENABLED` 显式开关，默认开启。

【Dev-A】WebSocket 原先匿名放行且任意来源，本轮改为登录态保护，并复用 CORS 白名单；日志 topic 的用户/任务隔离仍需下一轮继续。

【Dev-A】DataX 作业文件包含连接口令，本轮已改为随机作业文件名、运行完成清理、接口输出和执行日志脱敏。任务归属校验仍需补。

【Dev-A】外部进程工具原先先读输出再等待，超时可能失效；本轮已改为异步读流、先等待超时、超时强制终止，并新增回归测试。

### 5. 前端 UI

【Design】登录页存在设计令牌不统一、内联状态样式和图标体系混杂问题；本轮已清理登录页硬编码色、CapsLock 内联状态与字符警告。

【Design】`workspace.html` 样式加载顺序已改为设计系统最后加载，保证 `dn-design.css` 兜底。

【Design】旧弹窗、抽屉、z-index、断点和 JS 直接写 style 的存量仍较多，后续按页面逐步迁到 `DN.*` 组件和 CSS 状态类。

## 三、已完成修复

- 移除运行时配置、部署脚本、历史文档中的服务器地址、默认密码和 Doris 明文默认值，改为环境变量或占位符。
- Doris、DataX、Dolphin、Dashboard 相关默认连接值改为空配置显式失败。
- DataX 作业文件随机命名，运行输出和任务日志脱敏。
- 认证默认开启，WebSocket 握手要求登录态，WebSocket 来源复用 CORS 白名单。
- 数据模型审批队列 GET、同步错误行 GET 纳入敏感权限映射。
- 指标立即计算、项目新建按钮权限点对齐现有权限目录。
- 指标计算与 MDM 导入操作者改为服务端当前用户。
- `ProcessUtil` 超时控制修复并新增测试。
- 登录页和工作台样式加载顺序按设计系统收敛。

## 四、遗留风险

- TABLE ACL 还没有统一下沉到所有读数入口。
- 普通 GET 仍有“只登录不校验 view”的存量策略，需分模块收敛。
- WebSocket topic 仍需按用户、任务或会话隔离。
- DataX `/run` 仍需绑定任务归属，禁止按任意 jobDir 文件执行。
- Shell 任务仍需独立权限、审批、白名单或默认禁用策略。
- 全局异常 HTTP 状态码、SQL 错误脱敏、JDBC extraParams 白名单仍待处理。
- 信息架构文档与真实导航仍需同步。
