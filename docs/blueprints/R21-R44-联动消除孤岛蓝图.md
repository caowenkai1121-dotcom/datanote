> 架构师全系统普查: 126 功能, 39 孤岛。R21..R44 共24轮联动重构路线图。

## 孤岛清单(39)
- **健康分趋势研判/90天日历热力(总览内)** [governance]: 纯前端可视化死胡同, 趋势研判结论与日历格子均无点击下钻, 仅展示
- **成本排行(资产内)** [governance]: 纯展示死胡同, 成本Top热力与明细表无行操作/无下钻到资产详情
- **数据标准落地概览/成熟度(标准内)** [governance]: 纯前端聚合展示死胡同, 成熟度磁贴与简评无任何下钻链接
- **DCMM八大域成熟度自评(健康模块)** [governance]: 录入后仅本卡自展示, 与五维健康分/其它模块无数据联动, 自成闭环但孤立
- **操作时段热力/状态码分布(审计内)** [governance]: 纯可视化死胡同, 热力格与状态条均不可点击下钻
- **主题域管理(树+增删改)** [governance]: 主题树仅本模块自管, 不与资产/血缘/标准等关联(表未挂主题域), 无下钻去向, 是孤立的目录维护
- **主题域结构健康(主题域内)** [governance]: 纯展示死胡同, 提示不可点击, 且 tone 用了 warning/success/error 与组件 warn/ok/err 约定不符疑似无效
- **指标分类管理** [consumption-metrics（数据消费层 + 指标域）]: 孤岛: 「分类管理」tab 没有专属界面, 点了和「指标列表」共用同一面板, 仅刷新分类下拉; 纯展示且为死胡同, 无真正的分类管理功能
- **消费层概览（统计卡片）** [consumption-metrics（数据消费层 + 指标域）]: 纯展示死胡同: 5个统计数字均不可点击下钻到对应的新鲜度/僵尸/消费明细, 与同页其它卡片无交互联动
- **消费热度 Top（近30天调用条形图）** [consumption-metrics（数据消费层 + 指标域）]: 纯展示死胡同: 条形图不可下钻, 与指标驾驶舱/审计流水无点击联动
- **指标消费排行 Top20（按消费日志计数）** [consumption-metrics（数据消费层 + 指标域）]: 消费模块内为不可下钻的纯展示; 且与「消费热度Top」功能重复(同一消费日志的Top计数, 两个卡片并列), 联动价值低
- **僵尸指标（启用但从未取值）** [consumption-metrics（数据消费层 + 指标域）]: 纯展示死胡同: 识别出僵尸指标却无法就地处置(无计算/编辑/跳转入口), 闭环断裂
- **资产影响（改表反查消费它的指标）** [consumption-metrics（数据消费层 + 指标域）]: 纯展示死胡同: 从资产侧反查到下游指标后无法下钻到指标本身(消费驾驶舱/指标管理), 影响面只能看不能跳

# DataNote 消除孤岛 · 跨模块联动大蓝图（R21..R44）

> 【架构师·全盘】+【业务专家】联合产出。基于 workspace.html(16384 行) / gov-*.js / mdm-*.js 真实源码核对。核心结论：系统**功能已极其丰富但深链基础设施缺失**——`govGoModule(key)` / `mdmGoModule(key)` / `navigateTo(route)` 三个跳转入口**只收 key/route、不收上下文 ctx**，导致绝大多数"孤岛"本质是"渲染器收不到下钻参数"。打通它们 = 一次性盘活 30+ 个死胡同。

## 2) 【架构师·孤岛清单】逐个孤岛 → 接到哪 + 落地方式

> 前置改造（R21 完成）：把三大跳转入口升级为带上下文版本：
> - `govGoModule(key, ctx)` —— `switchGovModule(el, key, ctx)` 把 ctx 存 `window.__govCtx` 供渲染器消费
> - `mdmGoModule(key, ctx)` —— 同理 `window.__mdmCtx`
> - `navigateTo(route, ctx)` —— 存 `window.__navCtx`，各 view init 读取后清空

## 3) 【业务专家·价值评估】+ 端到端业务闭环链

**价值分级（基于真实生产场景）：**

- **高价值需强化联动（核心资产，必须双向打通）**：表元数据/资产目录、质量规则、治理工单、指标驾驶舱、血缘、同步任务、黄金记录、周期任务。这些是平台的"业务实体硬通货"，当前问题不是功能缺失而是**互相看不见对方**。
- **合理但孤立需接入**：Profiler探查、僵尸指标、资产影响面、MDM质量监控、系统监控大屏、手动/集成统计、消费审计、DCMM自评、主题域。功能成立但是"看得见摸不着"，接入后立刻产生价值。
- **伪需求建议弱化/合并**：指标分类管理(空tab)、成员权限矩阵(静态科普)、指标消费排行(与热度重复)、all-tables/favorite-check/sync online-offline(冗余接口)、dmSaveFieldMeta(死代码)。**建议删除或合并，减少认知噪音。**

**5 条端到端业务闭环链（重构后应能一气呵成走通）：**

1. **数据质量闭环**：资产目录/Profiler 发现高空值字段 →【一键建质量规则】→ 规则调度执行失败 →【自动生成治理工单】→ 工单指派负责人 → 复跑通过【自动关单】→ 健康分回升。  *(贯穿 assets→quality→health→工单)*
2. **指标可信度闭环**：消费驾驶舱看指标 →【输入质量】查来源表质量信号 →【点规则下钻质量详情】→ 来源表【血缘上游】定位脏数据源头 → 修复 → 重新计算指标。  *(贯穿 consumption→quality→lineage)*
3. **变更影响闭环**：数据地图改某表字段 →【消费指标Tab】反查下游指标 +【血缘影响面】反查下游表 →【指标可下钻驾驶舱】评估 →【一键暂停下游调度】→ 改表 →【重算指标/复跑质量】。  *(贯穿 datamap→consumption→lineage→scheduler)*
4. **敏感数据治理闭环**：分类分级采样识别敏感列 → 打标密级 →【数据集/指标查询自动脱敏】→ 消费审计留痕 →【安全态势】发现未脱敏策略 → 补策略。  *(贯穿 classification→security→consumption→audit)*
5. **主数据消费闭环**：建模定义实体属性(引用参考数据码表)→ 黄金记录录入 →【质量监控不合规→跳回修复】→【去重合并(存活规则)】→【变更审批落地】→【发布订阅推送下游】。  *(贯穿 modeling→golden→quality→dedup→approval→pubsub)*
