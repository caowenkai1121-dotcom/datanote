# 第89轮 —— 借鉴 hermes-agent:程序化技能记忆(skill = 可照做的操作手册)

> 承 R87/R88, 完成 hermes 借鉴三连的第三环。借鉴 hermes `skills/`(程序化记忆: SKILL = trigger + 有序步骤, 可被未来直接照做)升级我方自学习记忆: 原记忆只是【扁平事实】, 现让蒸馏器区分沉淀【经验(memory)】与【操作技能(skill)】, 技能以有序步骤入库、召回时作可执行 playbook 注入。

## 一、背景缺口
- `DnAiMemorySkill.type` 列早有(memory/skill), 但 `learn()` **恒写 "memory"** → 永不产生技能。
- 召回只渲染 `- 【title】content`, 把多步操作流程压成一句事实, 丢失"可照做"价值。
- hermes 洞见: 程序化记忆(skill)比陈述性记忆(fact)对 agent 更可执行 —— 同类任务下次可直接按步骤复刻。

## 二、落地(蒸馏分类 + 技能步骤入库 + 区分召回 + 前端徽标)

### 1. 蒸馏器 `buildDistillPrompt`(AiMemoryService)
JSON 增 `type` 字段, 明确二分:
- **type=skill**: 本次体现一套可复用多步流程(固定套路/标准做法, 如『建ODS表→建同步任务→配质量规则』), content 写成有序步骤 `1) 2) 3)…`, 每步含关键动作与注意点 —— 未来可直接照做的操作手册。
- **type=memory**: 仅一条结论/事实/坑, content 一两句即可。

### 2. `learn()` 落 type
- 读 `j.path("type")`, 仅取 {memory, skill}, 默认 memory(原硬编码 "memory" 移除)。
- skill content 上限放宽 600→900(容纳有序步骤)。
- 向量 payload 增 `type` 字段(便于未来按 type 过滤召回)。

### 3. `recall()` 区分渲染
- skill → `- 【操作技能·title】(适用: trigger) 1)…2)…3)…`(放宽呈现 320 字)。
- memory → `- 【经验·title】(适用: trigger) 事实`(200 字)。
- `RECALL_TEXT_CAP` 700→900 容纳技能步骤。
- PromptBuilder 记忆区头部改为"历史经验与操作技能(技能为可照做的有序步骤…)"。

### 4. 前端 `ai-agent.js` 记忆面板
- 每条记忆头部加 type 徽标:🛠 操作技能(靛蓝) / 🧠 经验(灰)。
- skill 的 content 用 `<pre>`(保留换行/缩进, 便于照做), memory 普通文本。

## 三、QA(本地连远程库 E2E — 真实产出技能)
触发会话:"在DataNote里排查同步任务是否健康应按什么步骤?用工具看现状再给可复用排查套路" → status=done(sync_dashboard 等工具链)。等异步蒸馏 ~12s 后:
- `/api/ai/agent/memories` 分组:**memory=20 skill=1**(type 流转鲜活, 非硬编码)。
- 新建技能(updatedAt 最新, hit=0):
  - 标题:**数据同步任务健康排查标准步骤** · **type=skill**
  - content(有序手册):`1) 调用 sync_dashboard 取全局快照, 关注状态分布(FAILED/STOPPED)/CDC任务数/调度在线数, 调度为零立即补全; 2) 对疑似异常CDC任务用 sync_job_detail 查详情(同步模式/状态/最后执行时间); 3) 对CDC任务用 sync_job_checkpoints 查位点积压, lag过大则降频或加资源。`
- → 蒸馏分类、技能步骤化入库、type 透传 API 全链路验真。下次同类任务 recall 即注入此 playbook。

## 四、安全不回退(记忆只读铁律保持)
- 技能仍是【只读上下文】, 绝不参与 Guardrail/ApprovalGate, 不得据此跳过审批/放宽护栏(防记忆投毒提权)。
- 蒸馏提示明确"绝不含密钥/口令/token"; 同租户隔离(owner)与去重(title)逻辑不变。
