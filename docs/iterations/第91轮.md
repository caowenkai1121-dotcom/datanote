# 第91轮 —— 智能体更聪明:反思自检(Reflexion, 证据对齐降幻觉)

> 承 R90(全服务点亮+知识增强感知)。本轮给 agent 加"反思"能力: 复杂(多工具)会话给出终答前, 先把答复草稿与已收集的工具证据严格对照, 修正未被支撑/矛盾的论断, 让结论可信、有据。借鉴 Reflexion 自我批判范式。

## 一、缺口
- 原主循环: LLM 一旦不再输出 tool_call 即把该段当终答直接返回, **无任何证据校验**。多步会话里, 模型可能在收尾时夹带未被工具证实的论断(幻觉)。
- 已收集的全部工具证据(trace)只用于"产出答案", 未用于"校验答案"。

## 二、落地(AiAgentService, 外科式)
- FINAL 分支: 终答草稿 `draft = sanitize(stripThink(raw))` → 经 `reflectIfNeeded(draft, …)` 得最终答, 再写 FINAL 步。
- `reflectIfNeeded`:
  - **门控**: 仅当 `toolCallCount(steps) >= 2`(有足够证据、值得反思)才触发; 单工具/无工具直接返回草稿, 省一次 LLM 调用。
  - 单次额外 LLM 调用(走 `callLlmWithRetry`, 复用同一 context 含『已执行步骤与工具结果』): "对照工具证据自检——有无未被支撑/矛盾的论断? 有则修正为只基于证据的严谨版本(可补关键依据), 已严谨则基本保持。直接输出最终中文答复。"
  - **保守**: 任何 AI 错误/空返回 → 退回原草稿(绝不因反思失败劣化答案); stripThink 剥净。
- 新增 `toolCallCount(steps)` 辅助。

## 三、QA(本地连远程库 E2E)
- **多工具(触发反思)**: "语义检索订单/支付相关表→挑一张血缘分析→能否安全下线" → status=done, 2 工具(semantic_search engine=vector + lineage_impact), 76s。终答严谨且**显式引用证据**: "下游依赖表数 0…**关键依据: lineage_impact 工具对 pay_center.payment_info 的查询结果**"。反思把结论与工具证据对齐并附依据。
- **单工具(跳过反思)**: 之前 issue_stats 类单工具会话 status=done 不变, gate 直接返回草稿无额外调用、无回退。
- 编译+打包+重启全绿。

## 四、权衡与安全
- **延迟**: 多工具会话 +1 次 LLM 调用(deepseek-v4-pro 较慢, 复杂会话总耗时上升到 ~70-80s)。已用 `>=2 工具` 门控把成本限制在真正复杂、最受益的会话; 简单会话零额外开销。
- **安全**: 反思仅重写终答文本, 不调用任何工具、不触碰 Guardrail/ApprovalGate/审计; 失败退回草稿, fail-safe。
