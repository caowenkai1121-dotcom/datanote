## 一、W5 cron 落地
### 表 + 模型
sql/72_ai_cron.sql 建 dn_ai_cron_job(name/schedule_cron/prompt/enabled/silent/owner/next_run/last_run/last_status/last_session_id/run_count), 远程 pymysql 已建。DnAiCronJob + Mapper。

### CronScheduler(@Scheduled fixedDelay 60s tick)
- 取到期任务(enabled=1 且 next_run<=now)。
- **先推进 next_run**(computeNext)再执行 + **原子占行**(UpdateWrapper 仅当 next_run 仍为旧值时推进, 抢到才跑)→ 崩溃幂等至多一次, 防重入/多实例重复。
- **算不出 next_run → 置 error 状态不静默禁用**(红线: 宁可报错也不静默停)。
- computeNext: everyNm/everyNh/everyNd + Spring 6 段 CronExpression。

### cron 无人值守模式(AiAgentService)
- runCron(prompt, ctx): ThreadLocal CRON_MODE=true → run() → finally 清理。
- cron 模式: manifest **隐藏 cron_job(防递归排程)/ask_user(无人应答)** + 主循环守卫拦截二者 + **不写自学习记忆**(防噪声)。
- 写操作仍走 Guardrail/ApprovalGate → cron 中遇写则挂起 wait_approval(安全 fail-closed, 不自动执行)。

### CronJobTool(agent 人话排程)
- action=create(name/schedule/prompt)/list/pause/resume/remove。护栏: 启用任务 ≤10、everyNm 最小间隔 ≥5 分钟、schedule 必须可解析、cron 会话内本工具被禁(防递归)。
- Controller: GET /crons, POST /cron/{id}/toggle, /cron/{id}/remove(可见性/管理)。
- PromptBuilder 协议第10条自治纪律。

## 天工开物 / 自由意志
- 天工开物: 定时是把工序按时间编排自动化; 先推进再执行(幂等)、算不出报错不静默(务实诚实)。
- 自由意志: agent 可自主用人话排程(自主), 但禁递归排程/无人时不 ask_user/写仍审批(护栏内自主), 任务可暂停/删除(可中断)。
