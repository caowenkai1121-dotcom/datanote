# 第55轮 —— 前端核心组件库 dn-common.js 深度优化（全 UI 共用，防御性加固）

## 本轮目标
深优**前端核心组件库 `dn-common.js`**（531L，停在 q100r57 迭代期，全部 gov*/mdm*/home/workspace 视图共用，是最高杠杆的前端文件）。逐组件审查空数据/空元素健壮性，**渲染行为不变**，仅消除脏数据致整页崩溃的隐患。

## 各角色产出

### 【Architect / Design】优化范围（DN.* 组件 5 处加固）
| 组件 | 优化点 | 说明 |
|---|---|---|
| `DN.h` | `setAttribute` 跳过 null/undefined 值（避免写出字面 `title="null"` 等脏属性）；`text` 属性 null→空串 | 正常属性渲染不变 |
| `DN.table` | 初始化与 `reload` 均过滤 null 行（`(o.rows||[]).filter(r=>r!=null)`），杜绝 `Object.keys(null)`/`r[k]` 搜索/排序/渲染崩溃 | 正常行数据/分页/搜索/排序/导出不变 |
| `DN.bars` | 剔除空项后再算 max 与渲染（防 `i.max`/`i.value` 取值崩） | 柱状渲染不变 |
| `DN.donut` | 段过滤加 `s &&`（防 null 段 `s.value` 崩） | 占比环渲染不变 |
| `DN.heat` | items 先过滤 null 再 map（防 `i.value` 崩） | 热力渲染不变 |

> 说明：上述均为「脏/空数据本会致整页 JS 崩」→「优雅跳过」的健壮性加固，正常数据路径零行为变化。版本 `q100r80`→**`q100r103`**（workspace.html 引用同步 bump，破缓存）。
