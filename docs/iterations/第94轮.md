# 第94轮 —— hermes 复刻 · 人在环交互: ask_user 决策/协助卡片(AskUserQuestion)

> 业主给参考图(tp/1.png 单选、2.png 多选)要求: agent 决策 / 需用户协助 / 审批时, 用【卡片提示框】让用户选择。复刻 hermes 的 AskUserQuestion 人在环交互, 注入自由意志·人机协同。

## 一、需求与设计
参考图 = Claude Code AskUserQuestion 卡片: N/M 分页徽标 + 问题 + 选项行(标签+说明, 单选带序号键/多选带复选框)+ Other 自填 + Skip/Next/Back/Submit。
设计: 新增 `ask_user` 工具 → 主循环【拦截即暂停】(status=wait_input, 回传 questions)→ 前端渲染卡片 → 用户答 → `/answer` 端点把选择注入为续跑消息, agent 续跑完成原任务(seed 在途上下文保连贯)。复用已有"暂停-恢复"骨架(同 approval), 弃 SSE。

## 二、落地
### 后端
- **AskUserTool**(name=ask_user, group=agent, readOnly 交互元工具不触审批/护栏): 参数 questions=[{header, question, multiSelect, options:[{label,desc}]}]。
- **AiAgentService.run** 拦截: 工具名=ask_user → 校验 questions 非空 → `st.awaitingInput=true / exitReason=AWAIT_INPUT / pendingQuestions=qs`, 写 ASK_USER 步, break。while 条件/grace/finalStatus/resp 全加 awaitingInput 分支(status=wait_input, resp 回传 questions)。
- **续跑保连贯 seedContinuation**: run 启动时取最近 FINAL 之后的在途工具步结果 seed 回 trace(ask_user 答后/中断后继续时不丢上下文)。
- **Controller `POST /{sid}/answer`**: formatAnswers 把 [{question,answer}] 拼成"(我已在卡片中做出选择)问→答…请据此继续"续跑消息 → run(sid, msg, ctx)。
- **PromptBuilder 协议第9条求助纪律**: 需用户拍板的关键选择/信息不足时调 ask_user 不臆断。

### 前端 ai-agent.js
- **questionCard**: 多问题分页(N/M 徽标); 单选=序号键(点选互斥)/多选=复选框(点选切换); 选中态靛蓝边框+焦点环; Other 自填输入(焦点环); 底栏 上一题/跳过/下一题|提交; 末题提交 → `POST /answer` → renderTurn 续渲染。
- **askingPreview**: 卡片上方"需要你确认："问题预览。renderTurn 遇 status=wait_input 渲染卡片(不出普通终答气泡)。

## 三、QA(本地连远程库 E2E + Playwright 真机视觉)
- **触发**: 让 agent 弹卡片 → status=**wait_input** exitReason=AWAIT_INPUT, questions=2(Q1单选 数据源类型 MySQL/PG/Oracle; Q2多选 同步特性 全量/增量/CDC)。
- **答案续跑**(轨迹铁证): [0]原请求→[1]ASK_USER→**[2]USER"(我已在卡片中做出选择)问:数据源类型…"(答案真注入)**→[3]semantic_search→[4]asset_detail(带PG/CDC上下文续跑)→[5]再追问(自主)。收尾 status=done 终答"基于 PostgreSQL+增量+CDC实时… create_sync_job 设 syncMode=CDC、源端开 wal_level=logical 建 publication+slot"——**跨多轮记住卡片选择+上下文连贯+可终止**。
- **Playwright 真机截图**: 卡片高度还原参考图 —— Q1 单选(1/2徽标+MySQL/PG/Oracle 标签+说明+序号键1/2/3+Other自填+跳过/下一题); Q2 多选(2/2+复选框☐+上一题/跳过/提交)。融入设计系统(靛蓝主色)。tools 35→**36**。

## 四、天工开物 / 自由意志
- **自由意志·人机协同**: agent 不臆断, 关键选择交业主拍板(ask_user); 业主在卡片中决策, agent 据此续作 —— 真正的人机协同。
- **天工开物·逐道工序透明**: 提问/回答/续跑各记一步(ASK_USER/USER/SKILL_CALL), 可回放; 选择注入轨迹可复核。
