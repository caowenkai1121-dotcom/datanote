# 第 12 轮 · 指标预警 — 阈值规则 + 指标值异常自动建治理工单（闭合 消费→治理 环）

> 接 R10/R11 消费层。新增**指标预警**：为指标配阈值规则，指标计算出值时自动判定越界，越界自动生成治理工单。
> 闭合"指标定义→计算取值→预警→治理工单→督办整改"全环，与质量→工单(R1)同闭环风格，复用 governance `IssueService`/工单表。

## 新增内容（domain.consumption）
- **表 sql/67_metric_alert.sql（幂等）**：`dn_metric_alert_rule`（metric_id, op[GT/LT/GE/LE/NE/OUT/IN], threshold_min/max, severity, enabled）。
- **MetricAlertService**：`checkAndAlert(metric, value)` 逐条启用规则判定 → 越界经 `IssueService.create` 建治理工单(issueType=METRIC, objectRef=`metric:{ruleId}` 去重，未关闭工单只刷新)；纯函数 `isBreach/breachDesc/normalizeSeverity` 可单测。全程兜底不抛。
- **钩子**：`MetricValueService.calc` 成功取得数值后调 `checkAndAlert` → 计算即预警（含定时计算 R11 自动触发）。
- **端点（ConsumptionController +3）**：GET `/metric/{id}/alert-rules`、POST `/metric/alert-rule/save`、DELETE `/metric/alert-rule/{ruleId}`。

## 闭环联动
`指标定义(governance.metric)` → `计算取值(consumption.MetricValueService)` → `阈值判定(MetricAlertService)` → `治理工单(governance.IssueService, 与质量工单同表/同流转)` → 督办整改。消费层成果反哺治理。
