# 第100轮 —— hermes 复刻 W6(部分): 后台复盘自我改进闭环

> Ultracode 续推。W6 取【后台复盘 background review】: 复杂会话完成后异步复盘执行质量, 找可改进点(绕路/冗余/遗漏/更优路径), 沉淀为 review 类记忆, 未来 recall 注入提醒"下次更好"。补齐 hermes 的自我进化层。

## 一、落地
- **AiMemoryService.reviewAsync**(@Async aiLearnExecutor): 复盘 prompt(找可改进点, 已高效则 worthSaving=false 不硬挑)→ 抽 JSON → 落库 type=review(title/trigger/pitfall, 全 redactSecrets 脱敏)+ 向量化(kind=memory, 同 learn 召回路径)。同 owner 同 title 去重 bump。
- **AiAgentService.run**: learn 之后, 仅【≥3 工具的复杂会话】(toolCallCount≥3, !cronMode)才触发 reviewAsync(选择性, 省 token)。
- **recall 渲染**: type=review → "【复盘改进·title】"(与 操作技能/经验 并列)。
- **前端记忆面板**: review 类徽标 🔍 复盘改进(琥珀色), 与 🛠操作技能/🧠经验 区分。

## 二、与既有自学习的区别(三层闭环)
- **learn(R67)**: 沉淀【成功经验/可复用技能】(什么有效)。
- **reflectIfNeeded(R91)**: 【当轮】终答对照证据自检降幻觉。
- **reviewAsync(R100/W6)**: 【会话后】复盘【可改进点/避坑】(下次怎样更好)。三者互补, 共同构成自我改进闭环。

## 三、QA(本地连远程库 E2E)
- 复杂会话(综合盘点治理: gov_overview/quality_score/sched_today_status 等 **7 工具**)status=done。
- 等异步复盘 → /memories `by type: memory=26 skill=12 review=1`; 日志 `[review] 复盘改进沉淀: id=39, title=遗漏计划内工具调用`(同时 learn 沉淀 skill id=38)。
- 复盘产出【真改进点】(识别出计划内工具遗漏), 非硬挑; 异步不阻塞主返回。

## 四、安全
- review 落库前全过 redactSecrets(脱敏红线); cron 模式不触发(不写记忆); 只读复盘不碰护栏/审批。
