## 二、本轮落地(P0): MDM 审批批准 → 真实应用变更
**根因**: MdmCoreController.review() 原仅 setStatus, 不动黄金记录 → 批准"删除"记录不删、"修改"不改 → 审批形同虚设, 与黄金记录直接编辑两条平行路。
**修**: 批准(approved)时先 applyChange(req) 再改状态:
- create: 新建黄金记录(dataJson=payloadJson, status=active, version=1);
- update: findGolden(优先 goldenRecordId, 退化 entityId+bizKey 未停用) → mergeJson 合并 payloadJson 进 dataJson, version+1;
- delete: 软删(status=inactive, 保留可追溯);
- 应用先于改状态: 应用失败(找不到记录/非法)抛错, 请求保持 pending 不误判; 回填 goldenRecordId 便追溯。
注入 ObjectMapper 做 JSON 合并。
