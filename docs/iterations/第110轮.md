## 一、下载链接不可点 — 真因+修(R109 没根治)
**真因**: agent 常把链接写成 `**[文件名](url)**`(粗体包裹)。renderMarkdown 的 INLINE_RE 中 `**粗体**` 在交替里排第一, 先匹配吃掉整段, 把 `[文件](url)` 当作粗体【纯文本】渲染 → 链接正则没机会命中 → 不可点。
**修**(前端 ai-agent.js):
- 粗体分支【递归】解析内联: `**...**` 内容再走 appendInline → 粗体内的 `[链接](url)`/裸下载URL 也渲染为可点 `<a>`。
- appendInline 用【每次调用独立的 RegExp 实例】(`new RegExp(INLINE_RE.source,'g')`), 防递归共享全局 lastIndex 串台。
- 验(Playwright 真机, pro+flash 双档): 导出 csv → 答复渲染蓝色可点 `⬇ 下载 xxx.csv`(href=/api/ai/agent/files/12/download, download 属性); 端点 HTTP200/text/csv/真实内容。对比图中纯文本, 已彻底可点。

## 三、空答兜底(顺带修)
**问题**: flash 偶尔只输出 `<think>` 或空 → stripThink 后终答为空 → 前端显示"（无答复）"(实测出现过 agent 看完数据没导出就空答)。
**修**: 主循环无 tool_call 分支, 终答为空时【再要一次】面向用户的明确终答(不计预算), 仍空则兜底文案; 并软化 trace 摘要为"答复不必逐行罗列; 若用户还要求导出/下载请继续完成"(防过度抑制导致漏做后续动作)。
