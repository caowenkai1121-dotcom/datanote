# 第113轮 —— 深挖:更智能工具 + 更快 + 诚实纪律(严禁臆造)

> 业主: ①工具更智能 ②速度更快 ③结果正确, AI不准乱猜, 不清楚即时求助, 不能胡编乱造。

## 一、③ 诚实纪律 / 反幻觉(本轮重点)
**协议第15条 诚实纪律(铁律)**:
- 库名/表名/字段名/数值/口径等具体事实【只能】来自工具证据, 严禁凭印象猜测编造; 工具没返回就如实说"未查到/暂无", 绝不杜撰看似合理的答案。
- 信息不足、库表名不明、多个可能对象需拍板时【立即 ask_user 求助】, 不替用户猜。
- 工具报错/结果空/数据异常/前后矛盾时【直接如实说明问题】, 不粉饰不绕过。
- 不确定就说不确定; 结论里每个具体事实都要能在工具结果找到出处。

**Reflexion 反幻觉强化**(reflectIfNeeded, ≥2工具时触发): 自检提示改为"逐一核对草稿里每个库名/表名/字段名/数值是否在工具结果有【出处】, 找不到出处的一律删除或改'未查到/不确定', 严禁保留臆造; 关键信息缺失就如实告知缺什么/建议怎么补/提示 ask_user"。

## 二、① 工具更智能 —— "表未找到"返回真实候选(防臆造)
- AssetDetailService.findSimilarTables(db,table): 从 dn_table_meta 找【真实】相近表(同表名→表名模糊→同库其它表)。
- AssetDetailTool: 表元数据为 null(不存在)时, 返回 not_found + 真实相近表清单, 提示"确认库表名或 ask_user, 切勿臆造"。
- TableDataTool: sampleRows 抛"does not exist/unknown table/不存在"类错时, 返回 not_found + 真实候选; 其它错误如实回传(不冒充 not_found, 守诚实纪律③)。
