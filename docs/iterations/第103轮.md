# 第103轮 —— agent 页面优化: 数据中心(文件上传/下载)+ AI 导出可下载文件

> 业主给参考图(企业智脑: 左数据中心上传/已上传文件 + 右AI助手), 要求【用户可自由上传文件 + 下载文件】。本轮把 agent 页面改为 2 列布局 + 全套文件上传/下载/列表/删除 + AI 生成可下载文件(复刻参考图的 AI 导出 CSV)。

## 一、布局(参考企业智脑, 2 列)
ai-agent.js build() 重构: #aiAgentRoot 改 flex-row = 左【📚数据中心】(width 300) + 右【聊天列】(head+对话流+输入)。抽屉(经验/定时/审批)仍绝对定位覆盖右侧。

## 二、文件上传/下载/列表/删除(后端)
- 表 `dn_ai_file`(sql/73, 远程 pymysql 已建): file_name(原名)/stored_name(uuid.ext 防穿越)/content_type/size/owner/source(user|agent)/session_id。
- **AiFileService**: save(MultipartFile)/saveContent(bytes)/list/resolve/delete。安全: 扩展名白名单(xlsx/xls/csv/pdf/txt/json/docx/md/图片, 无可执行)+ 20MB 上限 + 磁盘存 UUID(不用用户名作路径)+ 下载路径校验在存储根内(防穿越)+ owner 作用域。存储根 @Value `datanote.ai.file-dir`(默认 ai-files, 已 .gitignore)。
- **AiFileController** `/api/ai/agent/files`: POST /upload(多文件 MultipartFile)/ GET 列表 / GET /{id}/download(ResponseEntity<Resource> + Content-Disposition RFC5987 兼容中文名)/ POST /{id}/remove。owner 作用域。
- application.yml multipart 上限 20MB/60MB(可配 env)。

## 三、AI 生成可下载文件(复刻参考图)
- **export_file 工具**(group=agent, readOnly 仅写文件存储非业务数据): 参数 fileName(带扩展名)+content(文本), 经 AiFileService.saveContent 存为 source=agent 文件, 返回 downloadUrl。
- PromptBuilder 协议第11条导出纪律: 用户要导出/下载(csv/报表)时用 export_file 给下载链接, 不把大段 csv 铺答复里。
- send() 成功后 loadFiles() 自动拾取 agent 导出的文件到数据中心。

## 四、前端数据中心(ai-agent.js)
- 上传区: 点击 + 拖拽(dragover/drop 高亮), 隐藏 <input multiple>, fetch FormData 上传。
- 已上传文件: 列表(📄用户/🤖AI生成 + 文件名 + 大小)+ ↓下载(a href→download 端点)+ ✕删除。
