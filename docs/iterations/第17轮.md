## 角色

- **【Architect】** 主导。MDM 11 controller 多为 1-7 端点的碎片（Steward 仅 1 端点、Match/Quality 仅 2），职责可按内聚归并：核心实体+审批+管家 / 黄金记录+存活+匹配 / 参考+层级+交叉引用 / 质量+发布订阅。归并降低文件碎片、统一域内依赖，行为不变。
- **【Domain】** MDM 此前判定伪需求偏多，但既未冻结则**控制器碎片化**应整治；合并不增删功能、不改路径，用户与下游零影响，认可。
- **【Product】** 边界：4 controller = MdmCore(16端点) / MdmGolden(13) / MdmReference(14) / MdmGovernance(8)，合计 51 = 原 51。
- **【Dev-A】** 脚本化合并 `_merge_mdm.py`：解析各源 `@RequestMapping` 基路径→每方法映射改全路径、删类级基路径；注入 `private final` 字段取并集去重（Lombok `@RequiredArgsConstructor`）；保留 `objectMapper`/静态常量；同名 handler 改名（见下）。删 11 旧、写 4 新。
- **【QA】** 见下。
