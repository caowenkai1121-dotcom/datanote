- **【Domain】** 此前断裂的最后一公里：R15 端点能反查影响面，但用户在数据地图浏览表时看不到。数据工程师下线/改字段前，最自然的动作是在数据地图打开那张表——影响面就该出现在那里，而非另开消费看板手输库表。本轮补上，场景闭环。
- **【Architect】** 纯前端联动：表详情 Tab 复用既有 `dmSwitchTab` 机制，调 R15 已有端点，无新端点、无后端改动、无表变更。数据地图(metadata 视图)消费 consumption 端点——视图层跨域调用，不在 Java 层引入新依赖。
- **【Product】** 功能边界：表详情页 Tab 序列末尾加「消费指标」，懒加载 `asset-impact(db,table)`，列「指标/编码/关联类型/最新值/取值时间」；空态文案明确「改动此表对指标无已知影响」。
- **【Design】** 复用 `result-table` 表格样式 + 既有 Tab 视觉，与「分区/数据预览」等 Tab 一致，零新样式。
- **【Dev-B】** workspace.html：①表详情 Tab 栏加「消费指标」按钮；②`dmSwitchTab` 加 `impact` 分支；③新增 `dmLoadImpact()`（fetch asset-impact → result-table 渲染，含空态/错误态）。
