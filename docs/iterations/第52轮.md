# 第52轮 —— 编排层 + common 层服务深度优化（含 SQLi 加固 + 级联删 bug 修复）

## 本轮目标
深优编排层（补数据/SQL血缘/DolphinScheduler集成）+ 资产详情 + common 层（主题域/分组）共 6 服务。**重点：补 1 处 SQL 注入加固 + 修 1 个级联删除隐藏 bug + 1 处逆序区间脏数据 bug**，其余防 NPE/入参校验，**行为语义不变**。

## 各角色产出

### 【Architect / Domain】优化范围（6 服务）
| 服务 | 优化点 | 级别 |
|---|---|---|
| **AssetDetailService** | **SQL 注入加固**：`profile()` 把 `db`/`table` 直接拼入原生 `COUNT(*)`/聚合 SQL，原无校验 → 加 `IDENTIFIER_PATTERN` 校验（库名/表名拼前 `requireIdentifier`，列名拼前 `validIdentifier` 不合法跳过并标 error）；`assetDetail` columns 判空；`saveTerm` 入参判空 | 🔒 安全 |
| **SubjectService** | **级联删 bug 修复**：`delete` 注释称"级联删除子节点"实则仅删一层，孙节点孤立 → 改 `deleteRecursive` 递归删整棵子树（同 ScriptService.deleteFolderRecursively 模式）；`listTree`/`buildTree`/`create` 判空 | 🐞 修 bug |
| **BackfillService** | **逆序区间 bug**：`createBackfill` 对 `endDate<startDate` 算出负 totalDays → 任务无实例却永 RUNNING；加起止判空 + 逆序拦截 `BusinessException`；`executeBackfill` instances 判空 | 🐞 修 bug |
| SqlLineageService | `rebuildFromScripts` scripts 判空 + 元素判空；`insertEdges` 的 `getReadTables`/`getColumnMappings` 判空 | 🛡 防 NPE |
| DolphinService | `onlineScript` 的 `existingTaskCode` 自动拆箱 NPE（缺则补 `genTaskCode`）；`buildTaskDefinition` 的 `scriptType.toLowerCase()` 判空；`genTaskCode` 的 data 数组判空 | 🛡 防 NPE |
| GroupService | `create`/`update`/`addMember` 入参判空 | 🛡 防 NPE |

### 【QA】结论 —— ✅ 通过
- 编译通过（新增 import：BackfillService/AssetDetailService 的 `BusinessException`，AssetDetailService 的 `Pattern`）。
- 全量回归：`mvn -o test` **518 run / 0 fail / 0 error / 1 skip** → **BUILD SUCCESS**。
- 安全核对：profile() 的 db/table/列名均经 `^[\w一-龥]{1,128}$` 校验后才拼 SQL，与 QualityService 注入防线一致。
- bug 修复核对：
  - SubjectService.delete 现递归到叶子，无孤立残留（事务包裹整棵删除）。
  - BackfillService 逆序区间提前抛错，不再产生永 RUNNING 脏任务。
