# 第88轮 —— 借鉴 hermes-agent:think 过程透明化(救活 think_content 死列)

> 承 R87。借鉴 hermes `trajectory.py` 的 scratchpad/think 分离(`convert_scratchpad_to_think` / `has_incomplete_scratchpad`)与 `think_scrubber`:让 agent 把决策推理写进 `<think>…</think>`,后端抽取入库(救活闲置的 `DnAiStep.think_content` 列)、从终答剥离,前端分离呈现 —— 契合"天工开物·逐道工序透明可复核"身份。

## 一、背景缺口
- `DnAiStep` 早有 `think_content` 字段+列, 但 `writeStep` 调用处 think 参数**恒传 null** → 死列。
- 推理与结论混在一起: 模型若在答复里夹带思路, 会污染终答呈现。
- `stepsToDto` 未透传 think → 即便入库前端也看不到。

## 二、落地(后端抽取/剥离/透传 + 前端分离呈现)

### 1. `AgentTextUtil`(纯函数, 已直测)
- 新增 `THINK` 正则 `<think>(.*?)</think>`(DOTALL+忽略大小写)。
- `extractThink(text)`: 抠首个 think 内文(经 sanitize), 无则 null。
- `stripThink(text)`: 剥除所有 think 块(终答呈现前调用)。
- `parseToolCalls` 兜底分支改用 `extractJson(stripThink(text))`: 先剥 think 再抠 JSON, 防推理里的花括号误判工具调用。

### 2. `PromptBuilder` 协议第 0 条
新增引导: "每次回复先用一句 `<think>` 写出简短思路(为何选此工具/已掌握什么/下一步), 仅作过程留痕不入终答"。

### 3. `AiAgentService`
- 每步 `callLlmWithRetry` 后 `String think = extractThink(raw)`。
- 工具成功步: `writeStep(... think ...)` 入库(原 null)。
- FINAL(主循环/grace/resume 三处): 终答改 `sanitize(stripThink(raw))` 剥净 think; FINAL 步亦带 think 入库。
- `stepsToDto` 新增 `thinkContent` 透传前端。

### 4. 前端 `ai-agent.js`
- `toolCard`: 工具卡内新增 "💭 推理" 行(虚线/斜体/弱色, 左竖条同组色), 仅 `thinkContent` 存在才渲染。
- `renderTurn`: 终答气泡前, 若 FINAL 步有 think → 渲染 `thinkBubble`(虚线斜体气泡), 推理与结论视觉分离。

## 三、QA
- **逻辑直测**(javac 编译产物跑真方法, ASCII 样本): `extractThink`→取出推理✓; `stripThink`→去 think 留 tool_call✓(HAS_TOOLCALL=true/HAS_THINK=false); `parseToolCalls` 带 think 仍正确抠出工具✓; 终答 `<think>x</think>answer:0`→`answer:0`✓; 无 think→null✓。
- **E2E**(本地连远程库): 4 工具链(sync_dashboard→sync_job_detail×2→sync_job_checkpoints→FINAL)status=done; `finalHasThinkTag=False`(无 think 泄漏); 多步预算正常。
- **模型局限(如实记录)**: 当前 DB 配置 `deepseek-v4-pro` 为推理模型, **不在 output 外露 `<think>`** → 该列对此模型留空。机制正确且无害, 一旦切换为会输出 `<think>` 的模型(或提示词被遵循)即自动填充。非缺陷。
