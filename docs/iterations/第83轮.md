# 第83轮 —— 抽屉表单全新设计系统(黄金记录等表单美化人性化)

> 业主: 现有抽屉页面不好看(例"新建黄金记录"很丑、不人性化), 需找到所有抽屉/弹窗页面全面优化美化。R76/R77 已统一抽屉/弹窗**外框**, 本轮重做抽屉**内容(表单)**——建一套可复用的美观人性化表单设计系统, 先落地最丑的黄金记录 + 全部 mdm 表单。

## 一、旧表单丑在哪(黄金记录为例)
扁平属性堆叠(一行一字段)+ 标签 12px 弱灰 + 必填仅 " *" 文字 + 提示是技术类型"STRING(128)" + 保存按钮跟内容滚走(长表单要滚到底才能保存) + 只有保存无取消。长、散、不友好。

## 二、新抽屉表单设计系统(可复用, app.css `.dn-*` + dn-common.js `DN.*`)
- **分组小节** `DN.formSection(title)`: 标题(靛蓝竖条 + 细分割线), 把长表单切成有节奏的几组。
- **字段** `DN.field(label, control, {hint, required})`: 清晰标签 + **必填红星** + 控件 + 人性化提示。
- **控件** `.dn-form-input/.dn-form-select`: 36px 舒适高度 + 焦点环(`--ring`) + placeholder 柔和 + disabled 态。
- **两列** `DN.formGrid2([...])`: 短字段并两列(窄屏自动单列), 减少纵向冗长更平衡。
- **sticky 底栏** `DN.drawer(title, body, footerNode)` 新增第 3 参 → 渲染 `.df` 贴底操作栏(永远可见不滚走); `DN.drawerFoot({okText,onOk,onCancel})` 出 取消(ghost)+主按钮, 带 busy()/reset() 态。

## 三、落地 mdm 全部抽屉表单
- **goldenForm(黄金记录, 业主例子)**: 属性按 schema 动态字段 → 分"属性信息·实体"组(必填红星 + 可读类型提示 readableType: STRING→"文本，最长 N 字符" + 关键字段标注), 来源/状态并入"记录元信息"两列组, sticky 取消/创建记录底栏。
- **domainForm/entityForm/attrForm**: 同样分组 + 两列 + 必填红星 + sticky 底栏(attrForm 分基本/取值默认/约束排序 3 组)。
- field/inp/sel 升级为 .dn-* 类 → 顺带美化所有 mdm 表单输入。

## 四、QA(Playwright 真机, 0 console error, 527 测试绿)
