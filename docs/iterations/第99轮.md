## 二、修复清单(按维度)
### 安全/护栏(HIGH 优先)
- **[HIGH] IDOR 越权隔离**: 会话端点仅存在性判断无归属校验 → AiAgentService 加 `ownerOk` + loadOrInit 命中他人会话抛 SecurityException(run 捕获返 blocked)+ resume 加 owner 校验。防接管/驱动/读取他人会话。
- **[HIGH] 子代理禁用集补 cron_job**: 防 cron 模式子代理绕过自排程递归。
- **[HIGH] cron_job 工具越权**: list/pause/resume/remove + Controller crons/toggle/remove 全加 owner 作用域(匿名看全部), 防跨用户枚举/删除他人定时任务。
- **[MED] 持久化脱敏红线**: writeStep(args/result)/appendTrace(trace 喂 learn/reflect/compress)/记忆蒸馏(prompt 入参+落库 title/content) 统一过 redactSecrets, 与压缩器一致。
- **[MED] redactSecrets 扩覆盖**: 原仅 key=value → 增 JSON 引号包裹/短值/JDBC连接串口令(://user:pass@)/bearer/PEM 私钥块/sk-·AKIA·ghp_·xox 前缀。直测 7 例全过无误伤。

### 并发
- **[MED] AiAssistService 配置 volatile 快照**: 4 散字段(运行期可被 /api/ai/config 重写)→ 不可变 AiConf + 单 volatile 引用原子发布; chat/isAvailable 读一次快照。消除多线程(主/子/cron/learn 并发 chat)可见性 + 跨字段撕裂读(provider/baseUrl/key 错配)。
- **[MED] 会话级条纹锁**: 同 sessionId 并发 run(前端双击 / chat 与 answer 竞争)→ 64 条纹 ReentrantLock tryLock, 抢不到返"执行中", 防 seq 竞态/状态覆盖。零新依赖。

### 正确性
- **[MED] resume 写失败防谎报 footer**(与 run 对称, 重放是真正落地路径尤需)。
- **[MED] resume 终态 clobber**: updateById(陈旧 session)→ 定向 UpdateWrapper 只更 status/budget/updated_at(防回写覆盖并发 steer_text/plan_json), 与 run 收尾一致。
- **[MED] cron 最小间隔扩到 6 段 cron**(原仅 everyNm), 连算相邻执行间隔 <5min 则拒。
- **[MED] seedPriorTrace 取最近 12 条**(原 ASC 误取最早), 降序取+反转回正序。
- **[LOW] grace exitReason** 区分 BUDGET_EXHAUSTED / HARD_CAP_EXHAUSTED; 消 EMPTY_EXHAUSTED 幽灵态。
- **[LOW] CONTEXT_OVERFLOW 提前返回**: callLlmWithRetry 不再对超窗做注定失败的同 context 退避重试, 交外层 forceCompress。

### NPE/资源
- **[LOW] risk() NPE**: 两处 tool.risk().name() 加 null 守卫。
- **[LOW] toggleCron 容错解析**: 非数字 enabled 不再 500。
- **[MED] aiChildExecutor CallerRunsPolicy→AbortPolicy** + runBatch 捕获 RejectedExecution 降级占位: 防池满时整段子循环压到父 servlet 线程并绕过超时。
- **[MED] 子代理墙钟自检 + 超时取消**: runChild 循环加 deadline+interrupt 检查, runBatch 超时 future.cancel(true), 防父放弃后子成僵尸占池。
- **[MED] cron tick 不阻塞全局调度器**: 新增 aiCronExecutor, 占行后异步执行 runCron, @Scheduled tick 快速返回, 防拖死 CDC/同步/对账等 @Scheduled。
- **[MED] run 墙钟预算 300s**: 防 LLM 挂起把 Tomcat 工作线程长期占满拖垮 Web; 超时直接兜底不再发 LLM, exitReason=WALLCLOCK_EXHAUSTED。
