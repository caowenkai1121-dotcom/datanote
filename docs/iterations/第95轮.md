# 第95轮 —— hermes 复刻 Wave 2: 上下文压缩(/compress) + 错误分类

> 无人值守自驱。承研究蓝图 Wave 2(P0 长任务续航)。GraphRAG/RAG/记忆注入 + 多步/续跑使 trace 膨胀, 本波加上下文压缩(防超窗)+ 错误分类(限流/超长/鉴权差异化恢复)。

## 一、落地
### ErrorClassifier(错误分类, 借鉴 hermes error_classifier taxonomy)
把 AiAssistService.chat 错误串映射为恢复动作: `RETRY / RATE_LIMIT / CONTEXT_OVERFLOW / AUTH / ABORT`(429/限流→RATE_LIMIT, context length exceeded→CONTEXT_OVERFLOW, invalid api key→AUTH, 未配置→ABORT)。

### ContextCompressorService(上下文压缩)
- **maybeCompress**(每轮 build 前调): trace 超阈值(@Value `datanote.ai.compress-threshold` 默认 12000)→ 压缩。
- **按字符比例压缩**(适配我方"少量长行"的 trace 结构, 非 hermes 的多消息结构): 压前 ~55% 经辅助 LLM 摘成结构化摘要 + 保后 ~45% 逐字; 换行处切不断行。
- **结构化摘要 prompt**: 逐字保留对目标关键的(事实/已建对象ID/结论/卡点)+ 去重 + **绝不含密钥** + 只输出摘要。
- **安全/健壮**: 摘要落库前 `redactSecrets` 脱敏(password/token/api_key→***REDACTED***); 防抖动(节省<10% 或距上次增长<3000 跳过); 摘要失败(LLM抖/限流)静默退回原文不丢中段。
- **forceCompress**: 反应式超窗用, 无视阈值/防抖强压一次。

### AiAgentService 接通
- 循环内 build 前 `maybeCompress(st, goal)`(就地替换 st.trace, AgentState.trace 改非 final + compressionCount/lastCompressedLen)。
- **callLlmWithRetry 分类器化**: AUTH/ABORT 不重试(无意义); RATE_LIMIT 加长退避(backoffMillis(2)); 其余一般退避。
- **反应式超窗恢复**: chat 报 CONTEXT_OVERFLOW → forceCompress 后重建 context 重试一次。

## 二、修复(首测未触发的真因)
首测 8 工具 trace 未触发压缩。根因: compress() 原按**行数**保护(需≥14行), 但我方 trace 是"每工具 1 长行"(8工具≈8行<14)→ compress 恒返 null。改为**按字符比例**切(压前55%/保后45%), 适配少量长行结构。

## 三、QA
- **纯逻辑直测**(javac 编译产物跑真方法): redactSecrets(password/api_key→REDACTED, 无密钥不变)✓; ErrorClassifier 5 类(429→RATE_LIMIT/context→CONTEXT_OVERFLOW/auth→AUTH/未配置→ABORT/其他→RETRY)全对✓。
- **压缩 E2E**(低阈值3000 -D 测试): 6+表体检报告会话 → 日志 `[compress] trace 4487 → 2387 字符(第1次)`(47%节省>10%应用), status=done 8工具, 答复仍连贯报告(压缩不丢上下文)。
- **默认阈值(12000)无回退**: 正常单工具会话 status=done exitReason=DONE; 8工具会话不触发(trace<12K), 行为不变。阈值 @Value 可配, 默认对 DeepSeek 大窗安全。

## 天工开物 / 自由意志
- 天工开物: 压缩是独立可复核工序(前后字符数入日志留痕); 务实最小(单辅助LLM调用+限次+防抖)。
- 自由意志: 长任务超窗自主压缩续作(韧性), 失败显式退回不静默丢信息(可观测), 摘要禁泄凭据(护栏内)。
