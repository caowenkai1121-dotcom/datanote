# 第 32 轮 · 发布订阅自动触发 + 扇出逻辑去重(后端真闭环, 大版本)

> R21..R44 第十二轮。架构师:MDM 发布订阅原是"半闭环"——黄金记录 publish 只改状态, 推送订阅靠 pubsub/publish 手动模拟。本轮:①抽 MdmPublishService 统一扇出(去重);②黄金记录发布自动触发扇出(真闭环)。

## 改动(后端 domain.mdm)
- **新 `MdmPublishService`**: `fanOut(goldenRecord, changeType)` —— 对实体下启用且订阅该变更类型的订阅写发布日志, 返回命中数; `normalizeChangeTypes` 归一。供两处共用, 消除重复。
- **MdmGovernanceController.publish**(/api/mdm/pubsub/publish): 内联扇出循环替换为 `mdmPublishService.fanOut`; saveSubscription 改用 service.normalizeChangeTypes; 删私有 normalizeChangeTypes(去重)。
- **MdmGoldenController.publish**(/api/mdm/golden/{id}/publish): 发布后自动 `mdmPublishService.fanOut(rec,"update")` —— 发布即推送, 不再依赖手动 pubsub。

## QA —— ✅ 通过(E2E 闭环实测)
编译通过; **518 全量回归绿**; 本地运行。E2E(golden id=1/entity 1): 插临时 active 订阅(update)→ POST golden/1/publish → code:0, 发布日志自动新增=1(自动扇出生效)→ MySQL 清理。手动 pubsub/publish 与自动发布共用同一 fanOut, 行为一致。
