# 第92轮 —— 加大力度:列级语义索引(异步) + GraphRAG grounding

> 业主"继续加大力度"。承 R90/R91 把向量库/图库用得更满更深: ①把**字段级**元数据(数千列)纳入向量库 → agent 能"语义找字段"; ②**GraphRAG** —— 语义召回的首张表自动图扩展其血缘邻居, grounding 带上关系网络。

## 一、R92 列级语义索引(向量库用满)
### 缺口
向量库此前覆盖 表/术语/指标/数据标准/词根, **缺字段级**。问"手机号字段在哪张表/金额字段有哪些"无法语义命中。

### 落地
- `VectorIndexService.indexColumns()`: 读 `dn_column_meta`, 一次性载入 表id→[db,name] 映射给每列带所属表上下文; 文本"字段 columnName(businessName) 属于表 db.table 类型… 说明…"; payload kind=column + db + table + name + title。pointId=UUID(column:id) 幂等。
- **异步化(关键)**: 列量大(数千~万), 同步重建会拖垮 HTTP(实测 >9.6min 客户端超时)。新增专用单线程执行器 `aiIndexExecutor`(AiAsyncConfig) + `reindexColumnsAsync() @Async` + volatile 状态(columnIndexing/lastColumnPoints)。端点 `POST /api/ai/store/sync-columns` 立即返回 started, 进度查 `GET /api/ai/store/sync-columns/status`; 已在跑则忽略重入。
- `SemanticSearchService.normalize` 加 `table` 字段(列需显所属表); `semantic_search` 工具描述/kind 已含 column。

### QA
- `POST /sync-columns` → started=true 立即返回, 后台串行跑。status indexing=true 进行中, **增量即可检索**:
  - kind=column "手机号" → `phone 表=user_center.user_info 手机号`、`legal_person_cellphone 表=xh_dms.t_customer 法定代表人手机`。
  - kind=column "金额价格" → `amount 表=xh_dms.t_display_material_detail 费用金额`、`amount 表=…t_display_fee_detail 陈列费金额`。
- 字段级语义检索带所属表上下文, 正是"不知道字段在哪张表时语义找"。

## 二、R93 GraphRAG grounding(向量召回 × 图扩展)
### 思路
原 RAG grounding 只给孤立的语义召回项。GraphRAG: 对语义召回的**首张表**, 用血缘图扩展其上下游邻居, 注入"血缘网络"行 —— agent 一上来即见关系网络, 而非孤立表。

### 落地
- `AiAgentService.buildRagText` 末尾追加 `topTableLineage(results)`: 取首个 kind=table 命中 → `LineageEdgeService.trace/impact` 拉上下游(各截 5) → 渲染 `↳ db.table 血缘网络: 上游[a、b] → 本表 → 下游[x、y]`。
- 列(column)命中在 grounding 中显示为"字段 name (表 db.table)"(buildRagText kind 分支 + normalize 透传 table)。

### QA(铁证)
让 agent **不调任何工具**、仅凭注入上下文答"public.pg_users 有无下游":
- 结果: **工具调用=0**, 答"**有**, 下游是 `test_db.pg_users_from_pg`", 且直接引用注入的"`test_db.pg_users_from_pg 血缘网络: 上游[public.pg_users] → 本表`"。
- → 向量召回(命中 pg_users 系表)× 图扩展(其血缘邻居)成功注入 grounding, agent 零工具即知关系网络。

## 三、安全/降级
- 列索引异步单线程, 失败/不可用静默返 0; pointId 幂等(重跑更新不重复)。
- GraphRAG 仅只读注入, 异常静默 null; 不碰 Guardrail/审批。
- 派生入向量/图不回写业务库, MySQL 仍为 SoT。
