## 本轮目标
转入**项目模块**（13 个 service 仅 ProjectService 于 R48 深优，其余自 R9 后未动，headroom 更大）。本批 6 个核心 service：消 N+1、写方法事务原子化、防 NPE、入口校验，**行为语义不变**。

## 各角色产出

### 【Architect】优化范围（6 服务，均自 R9 后未深优）
| 服务 | 优化点 | 红线 |
|---|---|---|
| ProjectOverviewService | **boundJobRuns 消 N+1**：原每个 SYNC_JOB 资产单独 `selectOne(LIMIT 1)` 查最新执行 → 改一次性 `in(jobIds)` 批量查 + 内存按任务分组（id 倒序首条=最新，等价原 LIMIT 1）；`releases`/`jobs`/`members`/`assets` selectList 判空 + 元素判空 | 计数自洽（success/failed/running/other/neverRun=total）、活动流取 15/最近 8 语义不变 |
| ProjectAssetService | **projectsOfAsset 消 N+1**：原每个绑定单独 `selectById` 查项目 → 改去重 `selectBatchIds` 批量 + 内存映射 + 保持原绑定顺序输出（同项目多绑按原语义重复）；新增 `nz()` 助手，`allOfType` 4 处 + `candidates` selectList 判空 | DELETED 项目过滤、候选 bound 标注、绑定/解绑校验语义不变 |
| ProjectHomeService | `home`：`list(null)`/`listFavorites`/`recent(8)`/`pending`/`tasks` 全判空 + 元素判空（项目已删跳过逻辑保留） | 我负责/收藏/最近/待审批/我的待办聚合语义不变 |
| ProjectCollabService | `createAnnouncement` 入参 `a` 判空（防 NPE on `a.getTitle()`） | 公告/邀请/加入审批状态机、`acceptInvite` 事务+synchronized 不变 |
| ProjectReleaseService | `nextVersion` 的 `rs` selectList 判空 | 发布状态机 `ReleaseState.require` 护栏、版本递增语义不变 |
| ProjectTemplateService | **createFromTemplate 加 `@Transactional(rollbackFor)`**：原 save 项目 + 多次 member insert 非原子，中途失败致项目建成而成员部分缺失 → 整体原子；`saveAsTemplate` member selectList 判空 | 模板快照不含敏感 env-params、owner 由新建者承担、成员去重语义不变 |

### 【QA】结论 —— ✅ 通过
- 编译通过（ProjectTemplateService 新增 `@Transactional` import）。
- 全量回归：`mvn -o test` **518 run / 0 fail / 0 error / 1 skip（Docker testcontainers）** → **BUILD SUCCESS**。
- N+1 等价性核对：
  - boundJobRuns：原「逐任务取今日 DbSync 最新一条」→ 批量 `in` 后 id 倒序遍历 `putIfAbsent`，每任务保留首条（=最新），请求数 N→1。
  - projectsOfAsset：原「逐绑定 selectById」→ `selectBatchIds(去重)` + 二次遍历 bindings 保序输出，重复绑定按原语义重复，请求数 N→1。
- 事务正确性：createFromTemplate 此前无事务，加 `rollbackFor=Exception.class` 仅增强原子性，无业务分支改动。
