# 第53轮 —— 集成 schema 层 + common 层服务深度优化（含类型溢出/广播隔离/清理隔离）

## 本轮目标
深优集成 schema 层（漂移检测/自动建表/类型映射）+ common 层（数据清理/日志广播）共 5 服务。聚焦边界溢出、辅助流程失败隔离、防 NPE，**正常路径行为不变**。

## 各角色产出

### 【Architect】优化范围（5 服务）
| 服务 | 优化点 | 级别 |
|---|---|---|
| **TypeMappingService** | **整数溢出边界**：`varchar(99999999999)` 等超大括号数字 `Integer.parseInt` 会溢出抛 `NumberFormatException` → 抽 `parseIntSafe` 兜底（lenOrDefault/precisionOrDefault 返 def，DECIMAL 精度兜底 38/0 再走既有上限裁剪） | 🐞 边界 |
| **LogBroadcastService** | **广播失败隔离**：4 处 `convertAndSend` 原会冒泡 → 抽 `send()` 统一 try/catch（broker 不可用/序列化异常只记日志），辅助通知绝不中断调用方业务流（任务执行/状态更新）；顺带去重 | 🛡 鲁棒 |
| **DataCleanupService** | **清理隔离**：3 张表清理原串行无隔离，单表失败(锁超时)阻断其余 → 每删独立 try/catch，本次失败的表下次再清 | 🛡 鲁棒 |
| SchemaDriftService | `checkAndTrack` 的 `cols` 入参判空 + 元素判空 | 🛡 防 NPE |
| TableSchemaService | `ensureTargetTable` 的 `listTables` 结果判空（`stream()` NPE）+ 元素判空 | 🛡 防 NPE |
