# 第96轮 —— hermes 复刻 Wave 3: 子代理委派与并行隔离(delegate_task)

> 无人值守自驱。研究蓝图 Wave 3(P0 分治并行)最复杂一项。让父 agent 把多个【只读探查子任务】分派给子代理【并行】执行, 各子隔离会话独立完成回小结, 父汇总。天工开物·工具链编排分治 + 自由意志·护栏内自主。

## 一、落地
### IterationBudget 并发安全
`int used` → `AtomicInteger`(CAS consume/refund), 支撑父与各子独立预算实例在并发委派下计数不错乱。

### aiChildExecutor(并行执行器)
AiAsyncConfig 新增 core2/max4/queue8 + CallerRunsPolicy(满则调用线程自跑不丢任务)。

### ChildAgentRunner(子代理执行器)
- **runChild**: 隔离新会话(goalIntent 标 `[子任务·父<sid>]`)+ 独立 IterationBudget(MAX_CHILD_STEPS=5)+ 受限只读循环(简化版主循环: build→chat→parse→只读工具 invoke→trace→直到小结/预算尽)+ 末尾 grace 小结。子步骤入 dn_ai_step 可回放。
- **runBatch**: CompletableFuture 提交 aiChildExecutor 并行(≤4), 各子独立超时 180s, 结果按提交序聚合。
- **安全铁律(fail-closed)**: 子工具集 = 父只读工具 **减去** {delegate_task, ask_user, todo} —— 子不能写业务数据、不能再委派(防递归, **深度=1**)、不能反向求助/规划; 任何非只读工具一律拒绝(子无权), 写动作仍由父层走审批。

### DelegateTaskTool(委派工具)
- name=delegate_task, group=agent, readOnly(子全只读→无写副作用→不触审批)。参数 tasks=[{goal}] 或字符串数组, 1~4 个独立子任务。
- invoke: 解析 goals(cap 4)→ childRunner.runBatch → 返回 {delegated, results:[{goal,status,summary,childSessionId}]}。

## 二、修复(循环依赖)
启动报循环依赖: AiToolRegistry(收集所有 AiTool 含 DelegateTaskTool)→ DelegateTaskTool → ChildAgentRunner → AiToolRegistry。打破: ChildAgentRunner 的 toolRegistry 改 `@Lazy @Autowired` 字段注入(惰性代理, 用时解析)。

## 三、QA(本地连远程库 E2E)
- 触发: "用 delegate_task 把3个只读分析(user_info字段画像 / t_customer字段画像 / 治理工单统计)并行交子代理, 汇总要点"。
- 结果: status=done 127s, delegate_task rs=ok, **delegated=3 并行子代理** 各返小结(子1 table_profile失败仍小结 / 子2 t_customer结构分析 / 子3 工单统计), 父汇总三者。tools 36→**37**。
- 隔离会话(各子独立 dn_ai_session/step)+ 只读受限(CHILD_BLOCKED 拒写/委派/求助)+ 独立预算 + 并行(CompletableFuture)全生效。

## 天工开物 / 自由意志
- 天工开物: 把大调研拆成可并行子工序流水线(多源并查), 父只看子最终小结不被中间过程污染(逐道工序透明分治), 务实(默认扁平 depth=1 + 并发≤4)。
- 自由意志: 父自主决定何时分治并行(自主规划), 子读写最小权限交集 + 写回父审批(护栏内自主), 子任务独立超时(可中断)。
