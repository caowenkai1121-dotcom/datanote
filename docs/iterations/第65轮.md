> 业主需求②: AI 能完成开发任务(建表/ODS/同步任务等)、写动作经真实 Service 持久化→对应模块可见、护栏审批、数据入库。架构师 Workflow 全局规划(写能力盘点→护栏/审批/可见性→蓝图)。本轮交付写框架 + 3 个 MEDIUM 写工具,打通"咽喉点唯一"安全链路。

## 一、写操作护栏框架(咽喉点: AiAgentService 循环内 validate 通过↔invoke 之间)
- **Guardrail**(纯静态可单测): 三态 PASS/DENY/NEED_APPROVAL,只认 `tool.readOnly()/tool.risk()/DENY_NAMES` 三不可变源(绝不受 args/memory/config 影响)。只读 23 工具 PASS 直放零回归;红线名(drop_table/grant_permission/save_credential/deploy... 二道防线)DENY;写工具 NEED_APPROVAL。
- **ApprovalGate**(状态机, dn_ai_approval 持久): 幂等键 session+skill+args(resume 重入按 args 稳定命中);HIGH 每次强审批,MEDIUM 会话级一次性豁免;首次挂 pending → 会话 wait_approval。
- **fail-closed 审计**: AuditService 外科新增 `recordReturning`(返主键)+`existsById`(回读);写工具 invoke 前先写审计+回读,未落库即拒执行;invoke 后补结果审计(发起+结果双流水)。
- **状态机**: running→(写)wait_approval→(approve)paused→(resume)running→done / (reject)blocked。AgentState 加 awaitingApproval;循环条件/grace/session.status 同步。

## 二、审批端点(AiAgentController)
- `GET /approvals?status=pending` 待审清单;`POST /approval/{id}/decide{decision}`(decided_by 服务端强制 + **禁自批** + 乐观 `where status=pending` 防竞态);`POST /{sessionId}/resume`(重入 run 命中已批点放行)。sql/71 加幂等唯一键 + 审批人/时间非空 CHECK。
