# 第105轮 —— xlsx 零依赖解析(agent 读 Excel 上传文件)

> 续 R104。参考图主格式是 xlsx, 但守"零新第三方依赖"红线(不引 POI)。用 JDK ZipFile + SAX 自研 xlsx→CSV 提取, 让 file_read 能读 Excel 内容供 agent 分析。

## 一、落地
- **XlsxTextExtractor**(零依赖): xlsx 本质是 zip → JDK ZipFile 读 `xl/sharedStrings.xml`(共享串表)+ 首个 worksheet, SAX 流式还原单元格 → CSV。取第一个工作表, 限 200 行/字符上限(概览分析非完整导出)。SAX 关 DOCTYPE/外部实体防 XXE。
- **AiFileService.readText**: ext=xlsx → 走 XlsxTextExtractor 返 CSV; 其余文本类直读; pdf/老式xls/图片仍提示不支持。
- FileReadTool 描述更新(xlsx 首表可读)。

## 二、关键修复(内联串)
首测 xlsx 字符串列全空(只出数字)。根因: openpyxl 把字符串写成【内联串】(t="inlineStr" 的 `<is><t>`)而非共享串, 而提取器只读了 `<v>`/共享索引。修: 单元格文本统一在 `<v>`(数字/共享索引) 或 `<t>`(内联串) 内捕获, `</c>` 时按 cellType 解析(s→共享表查索引, 否则取捕获文本)。直测: 表头+3行含菜名全出。

## 三、QA(本地连远程库 E2E)
- openpyxl 造 xlsx(菜品/销售额/销量 3 列 + 3 行)→ 上传 → 问"销量最高菜品" → file_read rs=ok, agent 完整读出菜名(高品质大串羔羊肉/酥脆肉夹馍/热汤刀削面)+销售额+销量, 识别销量最高 312🥇。
- 直测提取器: 表头 菜品,销售额,销量 + 数据行 高品质大串羔羊肉,948.49,312 等, 修复后菜名不再缺失。

## 安全
- 零新依赖(JDK ZipFile/SAX); SAX 禁 DOCTYPE/外部实体防 XXE; 沿用 file_read 的 owner 作用域 + 路径根校验; 限行/字符防大表撑爆。
