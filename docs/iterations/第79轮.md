# 第79轮 —— 排版精修(字形抗锯齿 + 数据数字等宽对齐)

> 设计深化收尾的"premium 细节":提升全站字形锐利度, 并让数据平台所有表格/输入/数值的数字列**等宽对齐**(tabular figures), 数据更专业好读。离线环境无 webfont, 故走 system-ui 原生最优路线。

## 一、改动(app.css, 全局一处)
- **字族**: `-apple-system, ...` → `system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Microsoft YaHei', 'Helvetica Neue', Arial, sans-serif`(system-ui 优先, 各平台原生字体最清爽; 离线零额外字体请求)。
- **字形锐利**: body 加 `-webkit-font-smoothing: antialiased` + `-moz-osx-font-smoothing: grayscale` + `text-rendering: optimizeLegibility`。
- **数字等宽**: 全局 `table, .iw-form-input, .dbsync-form-input, .g-modal-input, input[type=number], code, pre, kbd { font-variant-numeric: tabular-nums }` —— 所有表格/输入框/代码的数字按等宽渲染, 列对齐不抖动(只影响数字, 中文不变)。KPI 数值原已 tabular-nums。

## 二、规范同步(docs/设计规范.md §4)
更新字族栈 + 字形规则 + "所有表格/输入/代码/KPI 数值统一 tabular-nums(app.css 已全局生效, 新组件无需重复声明)"。

## 三、QA(Playwright 真机)
- body 计算字族首位 = `system-ui`; `-webkit-font-smoothing` = `antialiased`。
- 表格计算 `font-variant-numeric` = `tabular-nums`。
- 527 测试绿(CSS-only, 后端无改)。
