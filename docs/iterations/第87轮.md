> 业主调整方向:深研 NousResearch/hermes-agent 借鉴提升我方 AI Agent(天工司辰)。本轮取其**最高价值/最低风险**的三处循环健壮性模式,外科式植入 `AiAgentService` 主循环。后续 R88(think 透明化)/R89(程序化技能记忆)续接。

## 一、深研结论:hermes 模式 → 我方缺口

读 hermes 源码(`agent/iteration_budget.py` / `turn_retry_state.py` / `retry_utils.py` / `context_engine.py` / `turn_context.py` / `skill_*.py` / `trajectory.py`)对照我方 `AiAgentService`:

| hermes 模式 | 源文件 | 我方现状 | 缺口 |
|---|---|---|---|
| `IterationBudget` consume/**refund** | iteration_budget.py | `for i<MAX_STEPS=6` 死计数 | 解析错/未知工具/坏参 各吃 1 生产步, 废步蚕食真步 |
| `TurnRetryState` 一次性 recovery guard | turn_retry_state.py | 首次 `isAiError`→直接 blocked 全程中止 | 单次 LLM 抖动 = 整轮挂掉, **零重试** |
| `jittered_backoff` 去相关退避 | retry_utils.py | 无 | 重试无退避 |
| ContextEngine `protect_last_n` 压缩 | context_engine.py | trace 仅靠步数封顶 | (R88+ 处理) |
| 程序化技能 trigger+steps | skills/ | 记忆=扁平事实 type 恒 memory | (R89 处理) |
| think/scratchpad 分离 | trajectory.py | `DnAiStep.think_content` 列恒 null | (R88 处理) |

**hermes 核心洞见(iteration_budget.py 注释)**:`execute_code`(程序化/廉价 turn)通过 `refund()` 退还, 不吃预算 —— 即"只为生产性进展计费, 不惩罚可纠正的廉价步"。我方原 `MAX_STEPS=6` 把所有迭代(含模型格式错的废步)一视同仁, 真正能执行工具的步被废步蚕食。

## 二、落地(AiAgentService 外科式改造 + 新增 IterationBudget)

### 1. 新增 `engine/IterationBudget.java`(忠实借鉴, 极简 ~35 行)
`maxTotal/used` + `consume()/refund()/remaining()`。单会话单线程顺序跑, 刻意去锁(与 hermes 线程安全版区别, 从简)。

### 2. 主循环 `run()`:死计数 → 预算 while + 废步退还
- 常量:`MAX_STEPS=6` → `MAX_PRODUCTIVE_STEPS=8`(生产步上限) + `HARD_ITER_CAP=24`(总迭代硬顶, 含被退还废步, 防失控刷屏)。
- 循环头:`for(i<6)` → `while(budget.remaining()>0 && iter<HARD_ITER_CAP && !done…)`;进入即 `budget.consume()` 先占一个生产步。
- **可纠正废步 `budget.refund()` 退还**(三处, 模型格式错):① tool_call JSON 解析失败 ② 未知工具名 ③ 参数校验失败。退还后 `continue`, 不蚕食生产步。
- **非可纠正不退还**(刻意计费, 让其尽早收敛/止损):护栏 DENY(永久禁区)/ 审批 REJECTED / 写前审计失败。
- 成功执行工具 / 给出终答 → 保留消耗(正常计 1 生产步)。

### 3. LLM 一次性重试 + 抖动退避
- 新增 `callLlmWithRetry(userPrompt, context)`:首调 `chat` 若 `isAiError`(请求失败/空/格式异常), `backoffMillis(1)` 退避后**重试 1 次**;仍错才返回首个错误串。单次 DeepSeek 抖动不再直接中止整轮。
- 新增 `backoffMillis(attempt)`:借鉴 `jittered_backoff` —— `base800ms*2^(n-1)` 封顶 8s + `[0,0.5d)` 随机抖动去相关。
- 三处 LLM 调用统一走重试:主循环每步、达上限 grace 收尾、`resume()` 审批后收尾汇报。

## 三、安全不回退(护栏铁律保持)
- refund 仅作用于**模型格式错**的可纠正废步;**护栏/审批/审计** 三态判定与计费完全不变(DENY/REJECTED/audit_failed 照旧拒且不退还)。
- 重试只重发**同一** prompt/context, 不改写任何参数、不绕过任何审批门;写工具仍走 NEED_APPROVAL→挂起→人工批→resume 重放。
- HARD_ITER_CAP=24 兜底:即便模型连犯格式错, 总迭代封顶, 不会因 refund 无限循环。
