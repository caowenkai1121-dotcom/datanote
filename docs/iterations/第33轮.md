# 第 33 轮 · DCMM 成熟度自评汇入健康分(后端闭环, 大版本)

> R21..R44 第十三轮。架构师:DCMM 八大域成熟度自评原是"录入后只自展示"的孤立闭环, 不影响治理健康分。本轮把它作为第 6 维「组织成熟度」汇入五维健康分。

## 改动(后端 HealthScoreService)
- 新增维度 `DIM_MATURITY = "组织成熟度"`。
- `maturityScore()`: 取 DCMM 八大域最新自评(MaturityService.latest())score 均值(level 1-5 → ×20 兜底); 未自评给中性分 60。
- `dimensionDetail()` 加入组织成熟度维度; `dimensionWeights()` 用 `putIfAbsent(DIM_MATURITY, 10.0)` 确保即使配置表非空, 组织成熟度也始终有权重(否则不计入总分)。

## QA —— ✅ 通过(E2E 实测总分变化)
编译通过; **518 全量回归绿** + 34 健康相关单测绿; 本地运行。E2E: POST /api/gov/health/score/refresh 后 `/dimensions` 返回 **6 维含组织成熟度=60**, **totalScore 28.1→31.0**(组织成熟度参与加权使总分上升)——DCMM 自评不再孤立, 真正影响健康总分; 做了自评后该维度分随之变化。
