# 第48轮 —— 后端服务深度优化（第四批：项目/AI/审计/告警/调度/依赖）

## 本轮目标
延续 R45-R47 后端深优主线，对剩余 6 个核心服务做大范围深度优化：校验前置、异常归一、判空兜底、N+1 消除、防御性边界，**行为语义不变**。

## 各角色产出

### 【Architect】优化范围（6 服务）
| 服务 | 优化点 | 红线 |
|---|---|---|
| ProjectService | `getById` 加 id 判空 + `IllegalArgumentException`→`BusinessException` 归一 | CRUD/软删语义不变 |
| AiAssistService | 配置读取失败由静默吞错改为 warn 记录降级；空 userMessage 降级提示；响应解析全程判空（content/choices/error/text 节点）；nl2sql null schema 兜底 | NL2SQL/provider 多格式解析行为不变 |
| AuditService | `selectCount` NPE 兜底；`selectList`/`selectMaps` 结果统一判空（`safeMaps` 助手）；CSV 导出判空 | 审计完整性、CSV 格式不变 |
| AlertConfigService | `getByScriptId`/`save`/`calculateDelayThreshold` 入参判空 + `BusinessException`；`status` 硬编码 `1`→`DnSchedulerRun.STATUS_SUCCESS` 自释义；7 日窗口加 `LIMIT 1000` 防全表；**防时钟漂移**：endTime<startTime 的负耗时跳过不污染均值 | 延迟阈值算法语义不变 |
| SyncJobScheduler | 跳过无 id 脏任务记录避免 NPE；`depsReady` 上游 id 去重+剔空（`LinkedHashSet`）；**消 N+1**：逐上游 `selectOne` 改一次性 `in` 批量查 + 内存按上游分组取今日最新（id 倒序首条），语义等价 | 调度/CDC 行为不变、`DepReadyChecker.allReady` 未动 |
| TaskDependencyService | `scripts`/`syncTasks` 结果判空兜底；遍历元素 null/缺 id 防御；`allUpstreamsCompleted`/`InBatch` 入参 `BusinessException` 校验；上游 id 缺失跳过 | **环检测算法 + `@Transactional` 回滚未动** |

### 【QA】结论 —— ✅ 通过
- 编译：`mvn -o compile` **BUILD SUCCESS**（无编译错）。
- 修两处对抗审查发现的真问题：
  - **AuditService 缺 `import java.util.Collections`**（`Collections.emptyList()` 会编译失败）→ 补导入。
  - **ProjectService 多余 `@Transactional` 导入**（实际未使用）→ 删除，消无用导入。
- 全量回归：`mvn -o test` **518 run / 0 fail / 0 error / 1 skip（Docker testcontainers，本地无 Docker）** → **BUILD SUCCESS**。
- 常量核对：`DnSchedulerRun.STATUS_SUCCESS == 1`，`status` 替换前后**取值字节级一致**（仅自释义，非行为变更）。
- N+1 等价性核对：原逐上游取「今日 DbSync 最新一条（id desc LIMIT 1）」；新批量 `in` 后按 id 倒序遍历、每上游首次出现即最新 → **结果等价**，请求数由 N 次降为 1 次。
