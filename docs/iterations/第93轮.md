## 二、Wave 1 落地(P0, 零外部依赖, 复用已设计未落地的 DnAiSession 列)
关键发现: dn_ai_session 早有 **plan_json/steer_text/interrupt_flag** 三列(M3 设计未消费, sql/70 已建)。本波"接通已铺好的线"。

### 1. 显式 plan/todo 规划(自主规划透明化)
- 新增 **TodoTool**(name=todo, group=agent, readOnly 元工具不触护栏): action=set(写整份有序计划)/update(改某步状态)/get; 维护 dn_ai_session.plan_json。
- PromptBuilder 加协议第8条"规划纪律"(多步任务先 set 再逐步 update) + 8 参重载注入"# 任务计划"段。
- 主循环每轮取最新 plan_json 注入, run 响应回传 plan。前端 ai-agent.js **planCard 任务清单**(✓done/▶doing/○pending + 进度 done/total)。

### 2. 中途 steering + 协作式中断(可中断/人机协同)
- 主循环每轮边界查会话(替代 SSE 的 DB 标志位轮询): interrupt_flag=1 → 置 cancelled/exitReason=INTERRUPTED 立即停; steer_text → **原子排空**(UpdateWrapper where steer_text=cur 防并发丢失)并入 trace 作"用户中途补充指引"(不打断当前轮, 记 STEER 步)。
- Controller 加 `POST /{sessionId}/interrupt` 与 `/{sessionId}/steer`。

### 3. 退出原因诊断 + 写失败防谎报(透明可复核)
- AgentState 加 exitReason(DONE/BUDGET_EXHAUSTED/INTERRUPTED/BLOCKED/AWAIT_APPROVAL) 各退出点赋值 + 落库 INFO 日志; 响应回传 exitReason。
- 写工具 invoke 失败 → hadWriteFailure 留痕, 终答尾部如实追加 footer"⚠ 写操作X执行失败实际未生效"(对建表/同步有副作用工序尤关键)。

### 修复 2 处真 bug
- **ctx.sessionId 回填**: 新会话时 controller 传入 sessionId=null(真实 id 在 loadOrInit 内生成), 致 todo 工具拿不到会话失败 → run 里 `ctx.setSessionId(session.getSessionId())`。
- **终态更新 clobber**: 末尾 `updateById(session)` 用启动时陈旧 session 对象, 把 todo 写的 plan_json / drainSteer 清的 steer_text 按旧值回写 → 改定向 UpdateWrapper 只更 status/budget/updated_at。

## 三、天工开物 / 自由意志 注入
- **天工开物**: plan 把工具链编排显式化(逐道工序 pending→doing→done 透明可复核); 中断/转向在每道工序边界检查; 退出原因+写失败如实留痘(务实诚实)。
- **自由意志**: agent 护栏内自主规划(plan)并自跟踪; 业主可随时 steer 纠偏 / interrupt 叫停长任务(可中断 + 人机协同); exitReason/plan/footer 让全程可观测。

## 四、QA(本地连远程库 E2E, 全绿)
- **plan/todo**: 多步会话 agent 自主 todo set(3-4步)→执行→todo update(标 done); plan_json 持久化(`[{治理总览,done},{质量分,done},{改进建议,pending}]`); 响应回传 plan。tools 34→**35**。
- **interrupt**: 置标志再 chat → status=**cancelled** exitReason=**INTERRUPTED** 0工具 首轮即停。
- **steer**: 预置暗号 steer 再 chat → 答复含暗号XYZ(并入上下文) + STEER 步记录 + steer_text 排空(不被回写)。
- **exitReason**: 正常 DONE / 中断 INTERRUPTED 正确回传。
