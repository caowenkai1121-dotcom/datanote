## 二、根因 + 修复(6 处, 外科 + Playwright 真机验证)
| 图 | bug | 根因(实证定位) | 修法 |
|---|---|---|---|
| 3 | **全站主按钮隐形**(白底空白) | **redesign.css:276** 全站 `.btn-primary{background:var(--dash-brand-grad)}`, 但 `--dash-brand-grad` 仅在 `:is(#homeCockpit,#govModuleContent,#mdmModuleContent)` 内定义→**容器外变量未定义+无 fallback→整条 background 失效→透明→白字隐形**(浏览器 getComputedStyle 实锤透明) | 补 fallback `var(--dash-brand-grad, var(--primary))`: 容器内得渐变, 容器外得纯靛蓝, 全可见 |
| 1 | AI 欢迎区**巨型图标**撑满屏 | **dn-common.js DN.icon** 输出 `<svg viewBox>` 无 width/height→无 CSS 约束的容器内 SVG 撑大 | DN.icon 加 `width="100%" height="100%"`(属性, CSS 可覆盖, 填满容器盒) |
| 2 | AI 回复**原始 Markdown**(###/`|表|`/**) | ai-agent.js renderRich 只转深链 token, 不渲染 markdown | 加轻量 **renderMarkdown**(零依赖/XSS安全全 DOM 构造): 标题/粗体/行内码/代码块/有序无序列表/表格/引用/分隔/段落换行 + 保留深链 chip |
| 4 | 表格徽标**竖排**成 客\n户 | **gov-modern.css:66** `.gov-pill` 缺 `white-space:nowrap`, 窄列每字一行 | `.gov-pill` 加 `white-space:nowrap` |
| 4 | 卡片**等高拉伸**大片空白 | **gov-modern.css:17** `.gov-grid` 默认 `align-items:stretch`, 短内容卡被拉到与高卡等高 | `.gov-grid` 加 `align-items:start`, 卡片贴合内容高度 |
| 5 | 血缘**表单竖排错位** | **app.css:2980** `.iw-form-select{width:100%}` 使 picker 内每个 select 占满 100%→flex 放不下→换行竖排 | dbTablePicker 的 select 内联 `width:auto` 覆盖 100%, 库/表/按钮回到单行 |
