## 角色（Workflow 多代理）

- **【Architect】** 关键约束发现：`gov-modern.css` 的 `.gov-*` 组件类限定在 `:is(#govModuleContent,#mdmModuleContent)` 作用域，首页用 DN 组件拿不到样式 → 解法：首页挂载容器 `#homeCockpit`，并把该作用域选择器一次性扩展为 `:is(...,#homeCockpit)`(124 处)，零重写、与治理页不冲突。
- **【Design】** 视觉升级三件事：①宽屏化(解除 1100px → 1600px)；②品牌渐变 Hero 条 + 玻璃质感色边 KPI 卡 + 悬浮上浮；③`css/redesign.css` 纯叠加层，不改 app.css 全局 :root，只在 `#homeCockpit` 局部新增 `--dash-*` 令牌 + `.dash-*` 类。
- **【Domain】** 8 KPI + 12 卡片每个回答"谁在什么场景看"：数据负责人晨会一屏掌握资产体量/治理健康/质量/消费活跃/同步/待办闭环。C6/C7(新鲜度+僵尸)是消费层闭环临门一脚——把"指标建了没人用/数据放陈了"暴露到首页。
- **【Workflow】** 7 代理 / 82 端点摸排 / 627K tokens：并行 Understand(消费+质量 / 元数据+同步 / 项目+MDM+系统 / 组件库+CSS令牌+挂载点) → Design 综合蓝图 → 并行 Build(home-dashboard.js + redesign.css)。

## 经验教训
- **静态文件改了必 bump 版本号**：内容变但 `?v=` 不变 → 浏览器缓存命中旧文件(本轮 Hero 修复后忘 bump，截图仍淡，bump q100r86→q100r87 后正常)。
- **CSS 注释勿含 `*/` 字面**：`--text-*/...` 之类会提前闭合注释，吞后续规则。
