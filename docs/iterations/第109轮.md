## 一、借鉴 caveman(token 压缩哲学「few token do trick」)提升 agent
caveman 核心=输出/上下文 token 压缩。落地三处:
1. **答复简洁纪律**: PromptBuilder 协议加第12条「简洁纪律(caveman·少字达意)」——终答直给结论与关键数据, 去客套/铺垫/重复; 但【数据表格/下载链接/数值/字段名等具体信息必须完整保留, 不得为简洁而省略或编造】。
2. **工具清单 token 瘦身**: AiToolRegistry.toToolsManifestJson 去掉 group/risk 字段(LLM 选工具不参考, 审批由后端护栏强制)——每轮 prompt 省 token, 只留 name/description/readOnly/params。
3. (延续 R108)子代理默认走⚡快速档 + 精炼委派纪律, 同属 caveman「省」思路。

## 二、修下载链接不可点(根因+修)
**根因**: 答复气泡用 renderMarkdown 渲染, 但只支持 **粗体/码/标题/表格/列表**, 不解析 `[文本](url)` markdown 链接 → export_file 给的下载链接被当纯文本渲染, 不可点。
**修**:
- 前端 appendInline 的 INLINE_RE 增 `[文本](url)`(组7/8) 与 裸下载链接 `/api/ai/agent/files/\d+/download`(组9) → 渲染为真实 `<a href>`(urlChip); 下载类加 `download` 属性, 外链 `target=_blank`。
- PromptBuilder 协议第11条改: 下载链接必须用 **markdown 链接格式 [文件名](下载URL)** 给出才可点击。
- 验: 让 agent 导出 csv → 答复气泡渲染可点 `<a download href=/api/ai/agent/files/10/download>⬇ 销售订单_6行.csv`; 该端点 HTTP200/text/csv/Content-Disposition attachment/真实 CSV 内容。

## 三、会话内直接查看表数据(新功能)
**根因**: 原无"看真实数据行"的工具(table_profile 只给统计画像); 且 hiveConfig 只连数仓(Doris), 源库 xh_dms 不在其中("Database [xh_dms] does not exist")。
**修**:
- **AssetDetailService.sampleRows(db,table,limit)**: 数据源路由——MySQL 源库(dn_table_meta 有 datasourceId 且 dbType=MYSQL)经 **ConnectionManager**(Hikari 池)连源库取真实行; 否则连数仓。SELECT * LIMIT n(默认20上限50), 单元格>200字截断, 防注入校验标识符。
- **新工具 table_data**(只读 LOW): 返回 {db,table,returned,_preview:{columns,rows,source},_deeplink}。协议第13条引导"用户想看表实际数据时用 table_data"。
- **未截断 previews 通道**: dn_ai_step.result_data 有 16000 上限, 宽表(72列) JSON 会被截断致前端解析不出 → AiAgentService 用未截断的原始结果走独立 `previews` 字段回传。
- **前端数据表格卡 dataGridCard**: 表头 sticky + 横纵滚动 + 行号 + null 显 ∅ + 单元格省略; 常显不折叠。renderTurn 优先用 res.previews 渲染。
- 验(Playwright 真机): "用 table_data 看 xh_dms t_sales_order_his 6行" → 渲染数据表格卡 **73列(72字段+行号)×6真实行**(HJXH-DSO2026022600003/SO01/长沙红欢喜供应链…, source=源库 MySQL), 下方 agent 简洁小结。
