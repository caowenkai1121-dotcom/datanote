# 第 1 轮：高风险入口封堵与多角色审查修复

> 时间：2026-06-18  
> 本轮类型：全系统审查 + 外科式修复  
> 范围：前端 UI、后端安全、架构、业务流程、产品权限口径。

## 1) 【Domain】业务场景与判断标准

本轮优先保证“登录用户不能绕过安全边界读取或执行敏感能力”。判断标准：

- 未显式授权的敏感 GET 不应暴露审批、错误行等敏感数据。
- 审计操作者不能由请求参数伪造。
- 数据同步、指标计算、MDM 导入等生产动作应有最小安全兜底。

## 2) 【Architect】架构审查

发现配置来源分散、默认连接值不安全、WebSocket 与外部执行链路边界偏松。定调为：先封入口，再重构模块边界；本轮不大改架构，只移除可上线默认值、收紧默认认证和输出脱敏。

## 3) 【Product】产品审查

发现前端存在不存在的权限点、数据模型审批队列边界不清、IA 与真实导航不一致。本轮已修 `metrics:run`、`project:edit` 误用，并把审批队列入口加审批权限；IA 同步进入下一轮。

## 4) 【Design】UI 审查

发现登录页、工作台加载顺序、图标和状态样式不统一。本轮已处理登录页硬编码色、CapsLock 状态、字符警告和 `dn-design.css` 加载顺序；旧弹层、断点和 z-index 存量进入后续 UI 收敛轮。

## 5) 【Dev-A/B】实现结果

- 认证改为 `DATANOTE_AUTH_ENABLED` 显式控制，默认开启。
- WebSocket 握手要求登录态，来源复用 `datanote.cors.allowed-origins`。
- Doris/DataX/Dolphin/Dashboard 默认连接值移除，未配置时显式失败或显示未配置。
- 部署脚本和文档中的敏感值脱敏，部署脚本改从环境变量或 `deploy.env` 读取。
- DataX 作业文件随机命名，执行输出和执行日志脱敏。
- 同步错误行、数据模型审批队列 GET 加入敏感权限映射。
- `ProcessUtil` 改为异步读流 + 超时强制终止。
- 指标计算、MDM 导入操作者改为当前登录用户。
- 登录页和工作台 UI 按设计系统做最小修复。

- `node --check src\main\resources\static\js\datamodel.js`
- `node --check src\main\resources\static\js\project.js`

- 敏感字符串扫描：排除 `deploy.env`、`target`、`.git` 后，未发现本轮关注的服务器地址、登录口令、默认 Doris 密码残留。

受限：

- 应用内浏览器策略阻止访问 `file://` 与本机 `127.0.0.1` 静态预览，本轮未完成浏览器截图验证；已用静态语法检查、CSS/权限扫描替代。

结论：本轮高风险入口封堵通过；系统级彻底重构仍需继续。

## 7) 【PM-Lead】本轮汇总

**本轮目标**：完成多角色全系统审查，先修复可直接落地的高风险缺陷。

**各角色产出**：业务、产品、架构、UI、后端审查均已形成问题清单；开发已完成第一批安全和 UI 修复。

**QA 结论**：核心单测、编译、前端语法与敏感扫描通过；浏览器验证受策略限制。

**遗留问题**：TABLE ACL 全入口下沉、普通 GET view 权限、WebSocket topic 隔离、DataX job 归属校验、Shell 任务安全、IA 同步、DTO/API client 收敛、旧 UI 组件迁移。

**下一轮计划**：第 2 轮进入权限闭环，优先处理 TABLE ACL、`*:view` 读权限和项目角色硬拦。

## 8) 本轮续修：权限闭环与前端入口一致性

【Domain】脚本上线/下线属于生产调度变更，真实流程必须由申请人与审批人分离；开发人员只能提交审批，不能通过工具栏或命令面板直接绕过上线审批。

【Architect】本轮保持最小改动，不新增前端框架；把敏感输出脱敏下沉到通用工具 `SecretRedactor`，把前端权限入口统一收口到 `data-perm` 与命令面板 `perm` 元数据。

【Product】脚本工具栏“上线/下线”定位为提交上线/下线审批申请，权限点改为 `develop:edit`；审批队列仍由 `develop:approve` 控制；首页快捷入口、命令面板与按钮权限保持同一产品语义。

【Design】登录错误、旧消息框正文改为文本渲染，避免错误内容变成 HTML；主数据质量空态移除庆祝表情，保留克制、清晰的后台操作文案。

【Dev-A】完成 `CryptoUtil.decryptSafe` 密文解密失败 fail-closed、脚本直接上线拒绝、审批通过后本地上线、执行日志/广播/API 失败响应脱敏、项目资产候选/绑定 ACL、数据模型不可变列表过滤修复。

【Dev-B】完成登录页错误消息 XSS 修复、旧消息框正文 `textContent` 渲染、脚本上线按钮权限修正、命令面板无权命令过滤与执行前校验、首页快捷操作 `data-perm` 挂载、MDM 质量空态文案收敛。

【QA】已补充并通过 `CryptoUtilTest`、`ScheduleLifecycleServiceTest`、`ScriptApprovalServiceTest`、`SecretRedactorTest`、`LogBroadcastServiceTest`、`ProjectAssetServiceAclTest`、`GlobalExceptionHandlerTest`、`RTest`、`DataModelControllerTest`、`StaticFrontendSafetyTest` 等定向回归。当前阶段明确不部署，待本地全部修复与验证完成后再进入部署步骤。
