# 第104轮 —— agent 感知上传文件 + 动作实时显示

> 业主续: ①上传的文件 agent 要能感知(并读取分析)②agent 的动作要能显示(实时)。

## 一、agent 感知上传文件(Part 1)
- **文件清单注入**: AiAgentService.buildFilesText(owner) 把数据中心已上传文件(id/名/大小/来源)渲染, PromptBuilder 第9参 filesText 注入"# 已上传文件(需分析其内容时用 file_read(fileId) 读取)"。agent 首轮即知有哪些文件。
- **file_read 工具**(group=agent, readOnly, owner 作用域): 按 fileId 读文本类(csv/txt/json/md/xml/log)内容供分析(≤8000字符, 超截断); 二进制(xlsx/pdf/图片, 无解析库)返提示。AiFileService.readText 实现(白名单+路径根校验+owner)。
- **E2E**: 上传销售 csv → 问"读取分析最高销售额菜品" → agent file_read rs=ok, 答复正确识别"高品质大串羔羊肉 948.49 元、热汤刀削面 755…"。tools 42→**43**。

## 二、agent 动作实时显示(Part 2)
- **客户端预生成 sessionId**: 前端 send() 先 genSid()(crypto.randomUUID), 随 chat 上送; loadOrInit 改为【采用合法格式的客户端 sessionId】(否则服务端生成), 使前端发起即知会话 id。
- **运行中轮询**: send() 发起 chat(阻塞) 的同时, 每 1.5s 轮询 GET /session/{sid}, 把已逐步入库的 SKILL_CALL 步骤【实时渲染为工具卡】(seq 去重); chat 返回后停轮询、清实时卡, 由 renderTurn 统一重渲(plan/工具卡/think/终答)。替代 SSE(后端逐步 writeStep 入库, 前端轮询读)。
- **session 端点 owner 校验**: GET /session/{id} 加越权隔离(仅发起人可读轨迹, 匿名放行), 补 R99 IDOR 收口。
- **E2E**: 客户端 sessionId 被采用(返回==传入)+ session 端点可查步骤(轮询基座 OK)。前端运行中即见工具卡逐个出现。

## 安全
- file_read/buildFilesText owner 作用域; 文本白名单+路径根校验防穿越; 无解析库不触二进制。
- 客户端 sessionId 格式校验([0-9a-zA-Z]{8,64}); 命中他人会话经 ownerOk 拦截; session 读端点 owner 校验。
