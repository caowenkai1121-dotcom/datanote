# 第 22 轮 · 数据质量闭环 + 资产中枢交叉联动(大版本)

> R21..R44 第二轮。架构师定调"资产/质量规则为枢纽实体, 须双向打通"。本轮落地业务闭环链①(质量闭环)+②(指标可信度闭环)的关键边。

## 架构师·全盘
资产(表元数据)是平台硬通货, 此前各模块"互相看不见对方"。本轮以**资产为中枢**向质量/工单/血缘/敏感/消费五个方向打通; 并把 Profiler 探查 → 一键建规则, 让"发现问题"直连"治理动作"。

## 改动(多模块)
- **workspace.html**: 数据探查 Profiler 字段表加「建质量规则」列 → `dmBuildRule(col,dim)` → `navigateTo('governance',{gov:'quality',prefillRule:{db,table,column,dimension}})`(空值率高→completeness, 否则 uniqueness)。
- **gov-assets.js**: 资产详情抽屉顶部加「治理联动」按钮排 —— 质量规则/相关工单/血缘图谱/敏感分级/消费指标, 均 navigateTo 带 {table}。资产即枢纽, 一处直达五向。
- **gov-consumption.js**: 指标「输入质量」抽屉来源表/规则行可点 → `navigateTo('governance',{gov:'quality',table})` 下钻质量(只读变可处置)。
- **gov-quality.js**: 规则行动作扩为 看来源表血缘 / 数据地图 / 查看相关工单 —— 规则↔血缘↔数据地图↔工单闭环。

## 业务闭环(本轮打通)
1. 质量闭环: 数据地图 Profiler 高空值字段 →【建质量规则】预填直达 → (R21 已通)规则失败自动工单 → 健康分。
2. 指标可信度闭环: 消费驾驶舱→输入质量→【点来源表】下钻质量→规则→【看来源表血缘】定位脏数据源头。
3. 资产中枢: 资产详情一键到 质量/工单/血缘/敏感/消费 五向。
