# DataNote 信息架构 v3(IA v3)

> 2026-06-10 业主令:全局重审布局,报表太多太乱/功能用不到的删,授权自由合并/删除/新增菜单与模块。
> 本文件是菜单与模块结构的**唯一契约**,与 `docs/设计规范.md`(视觉)并列。新增菜单/模块前先读本文件。

## 一、设计原则

1. **一个职责一个家**:同一能力全站只在一个中心出现;发现重复,保留更完整的一侧,另一侧摘菜单。
2. **报表为行动服务**:不能导出下一步操作的纯观赏图表一律不上首页/总览;炫技可视化(热力矩阵/矩形树图)默认不做。
3. **摘菜单≠删代码**:低频模块摘除侧栏入口(`hidden: true`),渲染器与深链保留——跨模块跳转不断,需要时一行恢复。
4. **首页是驾驶舱不是橱窗**:首页只回答"系统健康吗/有什么要我处理/在跑什么"三个问题。

## 二、顶层导航(11 项,不变)

首页 / 数据开发 / 数据运维 / 数据地图 / 数据治理 / 主数据管理 / 数据同步 / 指标管理 / 项目管理 / 系统管理 / AI助手

## 三、本轮裁决清单

### 首页:12 卡 → 4 卡
保留:治理健康总分(主卡)/ 健康分趋势 / 最新治理工单 / 同步任务监控大盘。
删除:热门指标Top10、近24h质量执行、资产分布·按库、指标新鲜度·陈旧告警、待治理僵尸指标、系统资源监控(系统管理已有)、敏感字段最多的表、最近消费活动。
KPI 磁贴 8→7:删「近7天消费量」;「启用指标数」改跳指标管理。

### 数据治理:11 → 9 项
- **摘除「数据消费」**:指标驾驶舱/取值/僵尸指标与「指标管理」中心职责重复(hidden,深链可达)。
- **摘除「主题域」**:树管理低频,从资产目录深链进(hidden)。
- 报表删减:审计中心删时段/状态码双热力图;健康分删 DCMM 八大域自评卡;分类分级删敏感矩形树。

### 主数据管理:13 → 10 项
- **摘除「质量监控」**:与数据管家待办聚合重复(管家页留深链入口)。
- **摘除「存活性规则」**:它是匹配去重的子配置——「匹配去重」更名**「匹配合并」**,页内提供存活策略入口。
- **摘除「层级管理」**:静态树半成品,场景弱。

### 数据运维:7 → 5 项
- **摘除「数据集成」**:与数据同步中心整建制重复。
- **摘除「血缘管理」**:与治理-数据血缘重复。
