# M11 治理健康分 + 工单闭环 + DCMM 成熟度自评 — 实现计划

日期：2026-05-30　分支：gov-m11（基于 master 含 P0+P1）

## 一、目标

把治理本身的"健康分 / 工单 / 成熟度"补成闭环：
- 五维（规范/质量/安全/生命周期/血缘）加权打分 0-100，权重配置表可调，时序快照。
- 治理工单单一事实表：OPEN→FIXING→RESOLVED→VERIFIED→CLOSED 闭环 + 按 owner 路由 + 排行榜。
- DCMM 八大域成熟度自评问卷录入 + 雷达。
- 健康分大屏（总分 + 五维雷达 SVG 自绘）。

## 三、数据源盘点（各维度取数口径）

| 维度 | 来源 | 口径 | 缺失退化 |
|---|---|---|---|
| 规范 | dn_standard_check_run（M7） | 最近一次 pass_rate*100 | 无记录→中性分 60 |
| 质量 | dn_quality_run（M5） | 最近 N 次 SUCCESS 的 pass_rate 均值*100 | 无记录→中性分 60 |
| 安全 | dn_column_meta.security_level（M8） | 已分级列 / 总列 覆盖率*100 | 无列→中性分 60 |
| 血缘 | dn_lineage_edge + dn_table_meta（M3/M4） | 有血缘边的表 / 总表 覆盖率*100 | 无表→中性分 60 |
| 生命周期 | dn_table_meta（M10） | 有 owner+importance 的表 / 总表 *100（资产治理完整度近似） | 无表→中性分 60 |

中性分 60：数据源缺失时不惩罚也不奖励，且前端标注「数据源缺失/估算」。

## 四、表设计（sql/41_governance_health.sql 幂等）

1. dn_governance_metric：id, dimension(规范/质量/安全/生命周期/血缘), metric_code, metric_name, weight, enabled, 预置默认权重 规范20/质量25/安全25/生命周期15/血缘15（每维度一行兜底项，同维度内可再加项细分）。
2. dn_governance_score：id, score_date, total_score, dim_scores(JSON), created_at（时序快照）。
3. dn_governance_issue：id, issue_type, dimension, object_ref, title, description, severity, owner, status, created_at, updated_at（工单单一事实表）。
4. dn_maturity_assessment：id, domain(DCMM 八大域), score, level, note, assessed_at。

均 CREATE TABLE IF NOT EXISTS；metric 预置用 INSERT ... ON DUPLICATE KEY（uk(dimension, metric_code)）。

## 五、Java 文件清单（全新）

- model：DnGovernanceMetric / DnGovernanceScore / DnGovernanceIssue / DnMaturityAssessment
- mapper：对应 4 个 BaseMapper
- service：
  - HealthScoreService
    - **纯函数** `static double weightedScore(Map<String,Double> dimScores, Map<String,Double> weights)`：权重归一化（按出现的维度），缺维度跳过，结果 clamp 0-100，可单测。
    - `computeAndSnapshot()`：各维度取数（容错）→ weightedScore → 写 dn_governance_score 快照，返回明细。
    - `currentScore()` / `trend(days)` / `dimensionDetail()`。
  - IssueService：CRUD + transition（状态机校验合法流转）+ assign(owner) + leaderboard（按 owner 统计 open/closed）。
  - MaturityService：八大域自评 upsert（同 domain 覆盖最新）+ list（雷达数据）。
- controller：HealthController `/api/gov/health/*`
  - GET /score（当前最新快照）
  - POST /score/refresh（重算并快照）
  - GET /score/trend?days=30
  - GET /dimensions（五维明细 + 权重）
  - GET /issues（列表，可按 status/owner/dimension 过滤）
  - POST /issues（新建）
  - POST /issues/{id}/transition（状态流转，body: {status, operator}）
  - DELETE /issues/{id}
  - GET /issues/leaderboard（owner 排行）
  - GET /maturity（八大域列表）
  - POST /maturity（自评 upsert）

## 六、状态机（IssueService.transition）

合法流转：
- OPEN → FIXING, CLOSED
- FIXING → RESOLVED, OPEN
- RESOLVED → VERIFIED, FIXING
- VERIFIED → CLOSED, FIXING
- CLOSED → （终态，可重开到 OPEN）
非法流转抛 IllegalStateException（Controller 转 R.fail）。

## 七、单测（TDD 先红）

- HealthScoreWeightedTest（纯函数，无 Spring）：
  - 权重归一化：全维度 = 直接加权；
  - 缺维度：只按出现维度归一（权重重新分母）；
  - 边界：空 map → 0；分数 clamp（>100/<0）；全 0 → 0；
  - 等权回退：weights 缺某维度时该维度权重按 0 还是默认？口径：weights 里没有的维度权重取 0（即不计入），dimScores 里有但 weights 无 → 跳过。
- IssueTransitionTest（纯函数状态机 isLegalTransition）：合法/非法流转覆盖。

先写测试 → 跑红 → 实现 → 跑绿。
