# M12 全局审计中心 — 实现计划

> 范围：P2 / M12。设计依据 `docs/superpowers/specs/2026-05-30-data-governance-design.md` §七 P2 M12、`dn_audit_log` 表（§六 6.1 第 111 行）。
> 原则：极简、审计失败不阻断业务、Filter 零侵入（不改 WebMvcConfig / SecurityConfig）、只增不改既有 .java。

## 一、目标

为治理平台补齐统一审计能力：自动捕获变更类 API 操作（POST/PUT/DELETE）+ 登录，落 `dn_audit_log`（只增不改），提供全局检索（时间/类型/操作人/路径，分页）、按类型/人统计、CSV 导出；前端审计检索表格 + 导出。

## 二、表设计（`sql/42_audit.sql`，幂等 CREATE TABLE IF NOT EXISTS）

`dn_audit_log`：
- id BIGINT PK AUTO
- user_name VARCHAR(80)  操作人（匿名记 'anonymous'）
- action_type VARCHAR(20) LOGIN/DATA_ACCESS/EXPORT/PERM_CHANGE/META_CHANGE/RULE_CHANGE/LABEL_CHANGE/OTHER
- target VARCHAR(255) 操作对象（可空，预留）
- method VARCHAR(10) HTTP 方法
- path VARCHAR(255) 请求路径
- ip VARCHAR(64)
- status INT 响应状态码
- detail TEXT 详情（耗时/查询串等）
- created_at DATETIME DEFAULT CURRENT_TIMESTAMP
- INDEX(created_at), INDEX(user_name), INDEX(action_type)

只增不改：无 update 列、无触发器，仅 INSERT + 查询。

## 三、后端（全新文件，不动既有 .java）

1. `model/DnAuditLog.java` — MyBatis-Plus 实体，`@TableName("dn_audit_log")`，风格仿 `DnSyncJobAudit`。
2. `mapper/DnAuditLogMapper.java` — `extends BaseMapper<DnAuditLog>`。
3. `service/AuditService.java`：
   - `record(...)` 入库，try/catch 包裹，失败仅 log.warn 不抛（仿 sync `AuditLogService`）。
   - 纯静态函数（可单测，先红）：
     - `shouldAudit(method, path)` -> boolean：仅 POST/PUT/DELETE 且 `/api/**`；排除审计自身查询路径 `/api/gov/audit`（避免递归）；GET 不记。
     - `classify(method, path)` -> actionType：按路径关键字归类（login→LOGIN, /export→EXPORT, /rbac|/perm→PERM_CHANGE, /meta|/metadata|/lineage→META_CHANGE, /rule|/quality|/standard→RULE_CHANGE, /classification|/label|/confirm→LABEL_CHANGE, 其余→DATA_ACCESS/OTHER）。
     - `csvCell(s)` -> String：CSV 字段转义（含逗号/引号/换行用双引号包裹，内部引号翻倍）。
     - `toCsv(list)` -> String：表头 + 逐行拼接。
   - `search(...)` 分页检索（时间区间/类型/操作人/路径模糊），手动分页（count + QueryWrapper.last("LIMIT off,size")，无 Page 插件）。
   - `statByType()` / `statByUser()` 聚合统计（QueryWrapper groupBy / selectMaps）。
4. `filter/AuditFilter.java` — `@Component extends OncePerRequestFilter`：
   - Boot 自动注册 Filter bean，**不改 WebMvcConfig**。
   - 先 `filterChain.doFilter` 放行，finally 计算耗时；若 `shouldAudit` 命中，取 SecurityContextHolder 用户（匿名→anonymous）、method/path/ip/status/耗时，调 `auditService.record`。整段 try/catch，审计异常绝不影响主请求。
5. `controller/AuditController.java` — `@RequestMapping("/api/gov/audit")`：
   - `GET /search` 分页检索（参数 from/to/actionType/userName/path/page/size）。
   - `GET /export` CSV 导出（同过滤条件，写 HttpServletResponse，text/csv + 附件头，BOM 防中文乱码）。
   - `GET /stat/type`、`GET /stat/user` 统计。
   - 登录审计：提供 `POST /login-record`（前端登录成功后可显式调用）— 简化，避免改 AuthController。

## 四、前端（仅新增 `static/js/gov-audit.js`，governance.html 不改）

IIFE 注册 `window.GOV_RENDERERS.audit = function(c){...}`：过滤条（时间/类型/人/路径）+ 检索表格（分页）+ 导出按钮（直接 window.open export URL）。
同时在 `window` 暴露入口 `window.openGovAudit()` 供其它模块跳转（导航项后补，html 不改）。
全部走 `DN.*`（get/h/esc/toast）。
